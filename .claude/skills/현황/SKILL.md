사용자가 현재까지 만들어진 것들을 파악하고 싶어합니다.

다음을 조사해서 **비개발자가 이해할 수 있는 말로** 요약하세요:

1. **만들어진 화면들**: `frontend/src/app/` 아래 페이지 파일들을 읽고, 각 화면이 무엇인지 설명
2. **주요 기능**: 각 화면에서 할 수 있는 것들 (버튼, 탭, 목록 등)
3. **정책 문서 현황**: `docs/policies/` 파일들을 읽고:
   - 정리된 규칙 수
   - 미결정 사항 수 (체크박스가 비어있는 것)
4. **최근 변경**: `git log --oneline -10`으로 최근 작업 내역

출력 형식 (예시):
```
## 만들어진 화면
- 대시보드 (메인): 프로젝트 목록을 칸반 보드로 볼 수 있어요
- 강의 상세: 특정 강의의 일정, 정보, 영상 피드백을 볼 수 있어요
- 제작 요청: 새 강의 제작을 요청하는 폼이에요

## 정책 문서
- 정리된 규칙: 16개
- 아직 안 정해진 것: 15개 → /정책확인 으로 상세 확인

## 최근 작업
- 칸반 보드에 드래그앤드롭 추가
- 영상 피드백 탭 UI 개선
```
