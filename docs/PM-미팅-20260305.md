# PM 미팅 노트 — 2026.03.05

> 본격 개발 착수 전, 제작 요청 담당자(PM)와 미팅하여 문제 정의를 정교화하고 MVP 기능 범위를 확정한 내용.

---

## 1. 문제 정의 (정교화)

### 문제 1: PM 리소스 비례 증가

제작 강의 갯수가 증가함에 따라 PM 리소스가 비례하여 증가한다. 현재 수동 관리 방식으로는 강의 온타임 딜리버리 커버리지를 유지할 수 없는 리스크가 커지고 있다.

### 문제 2: 실시간 현황 공유 불가

커리큘럼기획 매니저, 운영기획 매니저, 튜터, 편집자, 검수자 등 유관계자들이 제작 현황 및 일정을 실시간으로 확인할 수 없다. PM에게 직접 물어봐야만 현황 파악이 가능.

---

## 2. 예상 성과

| 지표 | 목표 |
|---|---|
| PM 리소스 절약 | **주 7시간** |
| 제작 현황 공유 | 유관계자 **실시간** 확인 가능 |

---

## 3. MVP 페이지 구조 (확정)

### 페이지 1: 강의 제작 현황 대시보드

#### 1-1. 오늘 진행 강의

- 촬영, 편집, 검수, 리허설, 교안작성 등 **단계별로 오늘 강의들이 어떤 상태에 있는지** 실시간 확인
- **칸반(Kanban) 형태** 고려 중
- 핵심: "오늘 내가 뭘 챙겨야 하는지" 한눈에 파악

#### 1-2. 전체 프로젝트 리스트

- **마감일 기준으로 얼마 남았는지** 확인이 포인트
- 칸반 또는 리스트 형태 고려 중

#### 1-3. 강의 제작 일정 공유폼

- 기존에 PM 또는 커리큘럼기획 매니저가 작성하던 폼
- 기존: 폼 작성 → 노션 강의 제작 카드 자동 생성
- **변경**: 대시보드 상단 오른쪽 "강의 제작 요청" 버튼 → 모달/상세페이지에서 폼 작성 → 제출 시 프로젝트 리스트에 자동 추가

### 페이지 2: 강의 상세 페이지 (카드 클릭 시)

#### 2-1. 해당 강의 전체 일정

- 챕터별 촬영/편집/검수 일정을 시각적으로 표시

#### 2-2. 가이드 문서

- 튜터를 위한 **촬영 가이드**
- 편집자/검수자를 위한 **편집·검수 가이드**

#### 2-3. 일정 조정 기능

- PM이 일정을 직접 수정할 수 있는 기능

## 4. 기존 도구 및 연동 요구사항

### 현재 사용 도구

| 도구 | 용도 | 사용자 |
|---|---|---|
| **노션** | 일정 확인, 트래킹, 관리 페이지 | PM |
| **슬랙** | 소통 채널, 일정 알림, 상태 공유 | 전체 유관계자 |
| **구글 드라이브** | 강의 촬영본 적재 및 확인 | 튜터, 편집자, 검수자 |

### 시스템 전환 후에도 유지되는 것

- **슬랙**: 유관계자 간 주요 소통 채널은 그대로 슬랙 유지
- **구글 드라이브**: 강의 촬영본은 계속 구글 드라이브에 적재

### 추가 연동 필요사항

| 연동 대상 | 요구사항 |
|---|---|
| **슬랙 알림 자동화** | 태스크 상태 변경 시 해당 슬랙 채널에 자동 알림 발송 |
| **구글 드라이브 관리 자동화** | 촬영본 업로드 감지, 드라이브 링크 자동 연결 등 |

---

## 5. MVP 기능 확정 기준

문제 정의에서 도출한 두 가지 기준:

1. **PM 리소스 절약에 직접 기여하는가?** → 수동 반복 작업을 시스템이 대체
2. **유관계자의 실시간 현황 확인을 가능하게 하는가?** → 별도 문의 없이 시스템에서 직접 확인
