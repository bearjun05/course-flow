# 영상 피드백 상세 설계

> 강의 상세 페이지(페이지 2)의 `영상 피드백` 탭 기능 정의서.
> PM 미팅(2026-03-10) 및 영상 피드백 질문 리스트 셋업 문서 기반으로 작성.

---

## 1. 뷰 목적

강의 원본 영상을 PM과 커리큘럼 기획 매니저(커기매)가 각각 검수하고, 질문별 점수와 코멘트를 남기는 페이지.

- 영상의 **합격 / 재촬영** 여부를 판단
- 튜터에게 전달할 **피드백 문구를 자동 생성**
- 피드백을 **복사해서 슬랙 채널로 바로 전달**

---

## 2. 데이터 구조

```
프로젝트(강의)
└── 챕터(장) — CH1, CH2, ...
    └── 강(講) — 1-1, 1-2, ... (최소 관리 단위)
        ├── 영상 파일(들) — 한 강에 복수 영상 가능
        ├── PM 검수 — 6문항 점수 + 코멘트
        ├── 커기매 검수 — 6문항 점수 + 코멘트
        └── 판정 — 승인 / 보완 / 재촬영
```

### 강(講) 단위

- `1-1` = 1장(챕터) 1강, `2-3` = 2장 3강
- 강 수는 챕터별 예상 분량에서 자동 계산
- 모든 피드백·검수·판정은 **강 단위**로 진행

### 복수 영상 처리

한 강의 원본 영상이 여러 파일로 구성된 경우:
- 영상 플레이어 옆 **화살표 버튼**으로 이전/다음 영상 이동
- 현재 영상 재생 완료 시 **자동으로 다음 영상 재생**

---

## 3. 역할별 관점 분리

### PM 검수 — 영상 퀄리티

| 관점 | 설명 |
|---|---|
| 음성 전달력 | 발음, 볼륨, 잡음, 울림, 음량 편차 |
| 화면 구성 | 자막, 자료, 화면 전환, 강조 요소, 시선 흐름 |
| 촬영 품질 | 해상도, 조명, 프레이밍, 화면 깨짐, 싱크 |
| 진행 템포 | 속도, 끊김, 호흡 |
| 몰입감/완성도 | 어색함, 반복, 군더더기 |
| 편집 보완 가능성 | 재촬영 없이 편집으로 해결 가능한 수준인지 |

### 커기매 검수 — 영상 내용

| 관점 | 설명 |
|---|---|
| 학습 목표 반영 | 의도한 학습 목표가 영상에 충분히 반영되었는지 |
| 설명 정확성 | 개념 정의, 사례, 표현의 적절성 |
| 수강생 관점 이해도 | 난이도, 맥락 연결, 친절한 설명 |
| 예시 적절성 | 학습 내용을 잘 뒷받침하는지 |
| 커리큘럼 의도 반영 | 앞뒤 수업과의 연결성, 전체 흐름 |
| 범위 적절성 | 불필요하거나 빠진 내용 없이 적절한 범위인지 |

## 4. 플로우

```
튜터: 촬영 완료 → 영상 피드백 탭에 영상 업로드
  ↓
PM: 영상 시청 → 6문항 점수(1~5) + 타임스탬프 코멘트 입력
  ↓
커기매: 영상 시청 → 6문항 점수(1~5) + 타임스탬프 코멘트 입력
  ↓
시스템: 평균 점수 + 치명 문항 체크 → 자동 판정
  ↓
시스템: 판정 기반 피드백 문구 자동 생성
  ↓
PM: 자동 생성 문구 확인(필요 시 수정) → 복사 → 슬랙으로 튜터에게 전달
```

- PM과 커기매 검수는 순서 무관 (동시 진행 가능)
- 피드백란은 **튜터에게 비공개**
- 두 검수자 모두 완료해야 최종 판정

## 5. 질문 형식

### 점수 척도

| 1점 | 2점 | 3점 | 4점 | 5점 |
|---|---|---|---|---|
| 매우 미흡 | 미흡 | 보통 | 좋음 | 매우 좋음 |

### 텍스트 피드백

타임스탬프와 함께 구체적 개선 포인트를 작성한다.

- 예: `12:35` 화면 전환 어색함
- 예: `05:42` 예시 설명 부정확

## 6. PM 질문 (6문항)

| # | 질문 | 핵심 관점 |
|---|---|---|
| Q1 | 음성은 또렷하고 안정적으로 전달되는가? | 발음, 볼륨, 잡음, 울림, 음량 편차 |
| Q2 | 영상의 화면 구성과 가독성이 적절한가? | 자막, 자료, 화면 전환, 강조 요소, 시선 흐름 |
| Q3 | 촬영/녹화 품질이 일정 수준 이상인가? | 해상도, 조명, 프레이밍, 화면 깨짐, 싱크 |
| Q4 | 진행 속도와 호흡이 자연스러운가? | 빠르거나 느린지, 끊김 여부 |
| Q5 | 전체적으로 완성도 있는 시청 경험을 주는가? | 몰입감, 어색함, 반복/군더더기 |
| Q6 | 재촬영 없이 편집으로 보완 가능한 수준인가? | 실무적 판단 |

**치명 문항**: Q1(음성 전달), Q3(촬영 품질) — 2점 이하 시 평균 무관하게 보완/재촬영

## 7. 커기매 질문 (6문항)

| # | 질문 | 핵심 관점 |
|---|---|---|

| Q4 | 예시와 데모가 학습 내용을 잘 뒷받침하는가? | 적용 사례 |

| Q6 | 불필요하거나 빠진 내용 없이 적절한 범위로 구성되었는가? | 범위 적절성 |

**치명 문항**: Q1(학습 목표 반영), Q2(설명 정확성) — 2점 이하 시 평균 무관하게 보완/재촬영

## 8. 점수 기준 및 판정 로직

### 기본 평균 기준

| 평균 점수 | 판정 |
|---|---|
| **4.3점 이상** | 승인 |
| **3.5점 ~ 4.2점** | 보완 후 재검토 |
| **3.4점 이하** | 재촬영 권장 |

### 치명 문항 예외 처리

평균이 높아도 아래 문항이 **2점 이하**이면 바로 보완/재촬영:

- PM: Q1(음성 전달), Q3(촬영 품질)
- 커기매: Q1(학습 목표 반영), Q2(설명 정확성)

### 최종 판정 규칙

| 조건 | 판정 |
|---|---|
| 평균 4.3 이상 AND 치명 문항 모두 3점 이상 | **승인** |
| 평균 3.5 이상 BUT 일부 문항 2점 이하 | **보완 필요** |
| 평균 3.4 이하 OR 치명 문항 1~2점 | **재촬영 필요** |

## 9. 자동 생성 피드백 문구 템플릿

점수와 코멘트를 조합하여 **튜터에게 전달 가능한 자연어 문장**을 자동 생성한다.

구성: **총평 + 잘한 점(4~5점 문항) + 보완 필요 사항(1~3점 문항)**

### PM 문구 템플릿

#### Q1. 음성 전달력

| 점수 | 피드백 문구 |
|---|---|
| 5점 | 음성 전달이 전반적으로 매우 안정적이어서 수강에 불편함이 없습니다. 현재 수준을 유지해주시면 좋겠습니다. |
| 4점 | 음성 전달은 전반적으로 안정적입니다. 다만 일부 구간에서 톤이나 볼륨 편차가 있다면 조금만 더 다듬어주시면 좋겠습니다. |
| 3점 | 음성 전달은 전반적으로 무난하지만, 일부 구간에서 발음이나 볼륨이 또렷하지 않아 보완이 필요합니다. |
| 2점 | 음성 전달에서 불명확한 구간이 있어 수강 경험에 영향을 줄 수 있습니다. 발음, 볼륨, 잡음 여부를 중심으로 보완이 필요합니다. |
| 1점 | 음성 품질 이슈가 커서 현재 상태로는 수강 진행이 어렵습니다. 재녹음 또는 재촬영이 필요합니다. |

#### Q2. 화면 구성/가독성

| 점수 | 피드백 문구 |
|---|---|
| 5점 | 화면 구성과 정보 배치가 명확하여 학습 흐름을 따라가기 좋습니다. |
| 3점 | 화면 구성은 이해 가능한 수준이지만, 강조 요소나 자료 가독성 측면에서 일부 보완이 필요합니다. |
| 1~2점 | 화면 구성과 자료 가독성이 떨어져 학습 집중에 방해가 될 수 있습니다. 화면 설계 보완이 필요합니다. |

#### Q4. 진행 속도

| 점수 | 피드백 문구 |
|---|---|
| 5점 | 진행 속도와 호흡이 자연스러워 몰입감 있게 시청할 수 있습니다. |
| 3점 | 전체 진행은 무난하지만, 일부 구간에서 템포가 다소 빠르거나 느리게 느껴질 수 있습니다. |
| 1~2점 | 진행 속도와 호흡이 일정하지 않아 전달력이 떨어집니다. 템포 조정이 필요합니다. |

### 커기매 문구 템플릿

#### Q1. 학습 목표 반영 여부

| 점수 | 피드백 문구 |
|---|---|
| 5점 | 이번 수업에서 의도한 학습 목표가 영상에 충분히 반영되어 있습니다. |
| 4점 | 전반적으로 학습 목표는 잘 반영되어 있습니다. 다만 일부 메시지는 조금 더 선명하게 드러나면 좋겠습니다. |
| 3점 | 핵심 학습 목표는 담겨 있으나, 수강생이 명확하게 인지하기에는 보완이 필요한 부분이 있습니다. |
| 2점 | 학습 목표가 영상에서 충분히 드러나지 않아 의도 전달이 약합니다. 목표 중심으로 구조 보완이 필요합니다. |
| 1점 | 이번 수업의 핵심 목표가 충분히 반영되지 않아 재구성이 필요합니다. |

#### Q2. 설명 정확성

| 점수 | 피드백 문구 |
|---|---|
| 5점 | 설명이 정확하고 명료하여 오해 없이 학습할 수 있습니다. |
| 3점 | 전반적인 설명은 가능하지만, 일부 표현은 오해 소지가 있어 다듬는 것이 좋겠습니다. |
| 1~2점 | 설명 정확성 측면에서 보완이 필요합니다. 개념 정의와 표현을 다시 정리해주시면 좋겠습니다. |

#### Q3. 수강생 관점 이해도

| 점수 | 피드백 문구 |
|---|---|
| 5점 | 수강생 입장에서 이해하기 쉽게 설명되어 있어 학습 진입 장벽이 낮습니다. |
| 3점 | 설명은 가능하지만 초심자 기준에서는 일부 구간이 어렵게 느껴질 수 있습니다. |
| 1~2점 | 수강생 관점에서 설명의 난이도와 친절도가 다소 부족합니다. 더 쉽게 풀어주는 보완이 필요합니다. |

## 10. 코멘트 결합 로직

자동 생성 문구에 검수자의 타임스탬프 코멘트를 결합한다.

**결합 규칙**: `{점수 기반 문구}` + `특히 {코멘트 내용}`

**예시**:
- PM Q1 = 2점 + 코멘트 "3분 20초부터 배경 잡음이 커집니다."
- 결합 결과: "음성 전달에서 불명확한 구간이 있어 수강 경험에 영향을 줄 수 있습니다. 발음, 볼륨, 잡음 여부를 중심으로 보완이 필요합니다. 특히 3분 20초부터 배경 잡음이 크게 들립니다."

## 11. 최종 판정 문구

| 판정 | 문구 |
|---|---|
| **승인** | 전반적으로 의도한 내용과 영상 퀄리티가 충분히 확보되어 승인 가능합니다. |
| **보완 후 재검토** | 핵심 방향은 적절하나 일부 보완이 필요하여 수정 후 재검토가 필요합니다. |
| **재촬영** | 현재 상태로는 수강 경험과 내용 전달 측면에서 한계가 있어 재촬영이 필요합니다. |

## 12. 튜터 전달용 결과 예시

### PM 피드백 예시

```
전체적으로 영상의 기본 완성도는 괜찮지만, 일부 퀄리티 요소는 보완이 필요합니다.
음성 전달은 전반적으로 무난하나 몇몇 구간에서 잡음과 볼륨 편차가 느껴집니다.
특히 3분대 구간은 음성이 다소 작게 들려 조정이 필요합니다. 또한 화면 내
텍스트 가독성이 다소 떨어지는 장면이 있어 강조 요소와 화면 구성을 조금 더
정리해주시면 좋겠습니다. 진행 속도는 전반적으로 자연스럽습니다.
```

### 커기매 피드백 예시

```
이번 수업의 핵심 목표는 전반적으로 잘 담겨 있습니다. 다만 수강생 입장에서
일부 설명은 다소 압축적으로 느껴질 수 있어 보완이 필요합니다. 특히 예시가
개념 이해를 충분히 뒷받침하지 못하는 구간이 있어, 적용 사례를 조금 더
구체적으로 풀어주시면 좋겠습니다. 또한 핵심 개념 정의를 조금 더 명확하게
설명해주시면 학습 이해도가 높아질 것으로 보입니다.
```

## 13. 데이터 모델 (구현 참고)

현재 `src/lib/types.ts`의 `ChapterTask`에는 "강" 개념이 없다.
영상 피드백 구현 시 아래 타입이 추가로 필요하다.

```
Lecture (강)
├── id: string
├── chapterId: string        // 소속 챕터
├── lectureNumber: number    // 강 번호 (1, 2, 3...)
├── label: string            // "1-1", "1-2" 등
├── videoUrls: string[]      // 복수 영상 지원
├── pmReview: ReviewData | null
├── cmReview: ReviewData | null
└── verdict: "approved" | "revision" | "reshoot" | null

ReviewData
├── scores: { questionId: string, score: number }[]
├── comments: { questionId: string, text: string, timestamp?: string }[]
├── averageScore: number
└── completedAt: Date | null
```
