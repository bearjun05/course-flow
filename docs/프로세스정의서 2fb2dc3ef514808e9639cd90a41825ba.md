**🚨 기존 방식의 문제점**

1. 일정 산정 및 관리 페이지를 **PM이 단독으로 관리**
2. 작업 진행 여부를 **시스템이 아닌 사람이 직접 추적**
    - 상태 변경 시마다 수동 업데이트가 필요하여 운영 비용이 큼
    - 업데이트 누락 시 현재 진행 상태를 즉시 파악하기 어려움
3. 강의 제작 관련 작업 페이지의 분산
    - PM: 촬영 타임라인 관리
    - 튜터: 슬랙 채널 (바로가기 폴더, 제작 일정 캔버스 등)
    - 편집/검수: 외주작업 DB

✨ **개선 방향**

- 강의 제작에 참여하는 **누구나 쉽게 접근 가능**하며 **쉽게 편집이 가능**한 것을 기본 원칙으로 설정

    - 접근성과 수정 편의성을 높이고, 사람이 아닌 **시스템 중심의 진행 관리**로 전환

### 추후 우선순위

- **`P0` 정산 시스템 및 데이터 자동화**
    - **백오피스 로그 자동 기록**
        - 백오피스에 강의 교육과정의 상태가 제작 완료로 변경되면, 강의 제작 시스템 내 '최종 영상 분량'을 자동으로 업데이트
        - 튜터, 편집자, 검수자의 정산 기준은 백오피스 최종 영상 분량으로 진행됨
    - **정산 프로세스 자동화**
        - 시스템에 기록된 최종 분량과 사전에 합의된 단가(분당 단가 등)를 매칭하여 정산금을 자동 계산
        - 작업 상태가 '최종 완료'로 변경되면
            - 정산 담당자에게 데이터가 자동 전송
            - 작업자(튜터, 편집자, 검수자)에게 작업 정산금 안내 메일 자동 전송
- **`P1` 사용자 권한 관리 및 외부 연동**
    - **사외 인원 전용 권한 설정**
        - 튜터, 편집자, 검수자가 접속하여 함께 상태값을 수정할 수 있는 권한 부여
    - **외부 시스템 연동**
        - **EduOps 연동**: 튜터 구인 확정 시 제작 페이지가 자동으로 생성
        - **Slack 워크플로우**: 담당자가 시스템에서 '상태 변경' 클릭 시, 해당 슬랙 채널로 즉시 알림이 발송되어 별도의 보고 과정 생략 & 리마인드 알림 & 튜터가 슬랙 내에서 완료 버튼을 누르면 시스템의 상태가 변경
- **`P2` 제작 히스토리 및 산출 관리**
    - **이슈 트래킹**: 검수자와 편집자 간의 수정 요청 및 완료 피드백을 시스템 내에서 타임라인별로 기록 & 관리
        - 동영상 내에 마크기능을 삽입해 검수 결과 입력 (n분 m초: 편집 필요 등)
    - **강의 에셋 라이브러리**: 썸네일, 소스 코드, 최종 교안 등 제작 결과물을 통합 관리
    - **일정 템플릿**: 강의 롤아웃, 예상 분량에 따라 표준 일정과 체크리스트를 자동으로 세팅
