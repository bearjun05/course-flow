# 제작 요청 폼 정책

> 강의 제작 요청 시 입력 항목과 조건부 로직 정의

## 규칙

- **BZ-001** 제작 요청 폼은 10개 항목으로 구성
  - 필수: 강의명, 튜터 배정 여부, 사용처, 제작 유형, 롤아웃일, 강의 지급일, 예상 분량
  - 선택: 세부 일정 산정, 예상 챕터 수, 커리큘럼 초안

- **BZ-002** 튜터 배정 여부
  - O(배정 완료): 튜터 이름 입력 필드 표시
  - X(미배정): 자동으로 "구인 중"으로 판단
  - 적용 화면: 제작 요청 폼 2번 항목

- **DT-001** 사용처별 추가 입력
  - KDT 선택 시: 트랙명 선택 필수 (디지털 마케터, 그래픽 디자이너, 커머스 Spring, Kotlin&Spring, iOS, Flutter, Unity, 데이터분석, UXUI, PM, QAQC, Unreal, 단기심화 Spring)
  - KDC 선택 시: 추가 입력 없음
  - 기타 선택 시: 용도 텍스트 입력 필수
  - 적용 화면: 제작 요청 폼 3번 항목

- **DT-002** 제작 유형별 추가 입력
  - 신규: 추가 입력 없음
  - 리뉴얼: 부분/전체 선택 필수
    - 이전 강의명: 기존과 동일하지 않은 경우에만 입력
    - 부분 리뉴얼: 리뉴얼 범위 설명 입력
  - 적용 화면: 제작 요청 폼 4번 항목

- **PR-005** 세부 제작 일정 산정
  - "필요해": PM이 예상 분량과 롤아웃 기준으로 산정 → 튜터와 논의 후 확정
  - "내가 산정했어": 직접 산정한 일정 텍스트로 공유
  - 적용 화면: 제작 요청 폼 7번 항목

- **DT-003** 커리큘럼 초안
  - 있으면: 링크 공유
  - 없으면: 강의 컨셉 간단 설명
  - 최종 커리큘럼에는 각 장별, 강별 예상 분량이 반드시 포함되어야 함 (촬영 일정 산정 기준)
  - 적용 화면: 제작 요청 폼 10번 항목

- **SC-001** 롤아웃과 강의 지급일의 정의
  - 롤아웃: 내부적으로 산정한 최종 마감일 (출시일이 아님)
  - 강의 지급일: 실제 목표 출시일
  - 배경: 롤아웃은 내부 마감, 강의 지급일이 실제 출시 시점
  - 적용 화면: 제작 요청 폼, 프로젝트 상세, 대시보드 전체

- **BZ-003** 제출 완료 후 흐름
  - "PM이 빠른 시일 내에 내용 확인할게요" 메시지 표시
  - 프로젝트 페이지 바로가기 링크 제공
  - 적용 화면: 제작 요청 제출 완료 화면
