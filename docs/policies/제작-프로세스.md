# 제작 프로세스

> 강의 제작의 단계별 선후행 관계와 조건을 정의한다.

## 전체 흐름

제작 요청 → 환경 세팅 → 일정 산정 → 리허설 촬영 → 편집/검수 구인 → 본촬영 + 편집/검수 → 롤아웃

## 규칙

- **[PR-001]** 편집/검수 구인은 리허설 촬영 이후에 진행
  - 배경: 리허설 결과를 보고 촬영 방식이 확정되어야 편집 요건을 알 수 있음
  - 적용 화면: 프로젝트 상세 > 일정 탭

- **[PR-002]** 챕터별 공정은 촬영 → 편집 → 자막 → 검수 순서로 진행
  - 배경: 각 단계의 산출물이 다음 단계의 입력이 됨
  - 적용 화면: 프로젝트 상세 > 일정 탭, 간트 차트

- **[PR-003]** 촬영 완료 영상은 편집자에게 전달되며, 편집 완료 후 검수자에게 전달
  - 배경: 튜터 → 편집자 → 검수자 순서의 작업 핸드오프
  - 적용 화면: 프로젝트 상세 > 일정 탭

- **[PR-004]** 전체 장의 승인까지 모두 완료되면 롤아웃이 자동으로 "진행" 상태로 전환
  - 배경: 모든 챕터의 승인이 통과되어야 출시 가능
  - 자동 로직: 모든 장에 승인 태스크가 있고 전부 "완료"일 때 → 롤아웃 "대기" → "진행"
  - 적용 화면: 프로젝트 상세 > 일정 탭 > 강의 롤아웃 행

## 관리 범위

- 사업부: KDT, KDC, 신사업
- 제작 유형: 신규 제작, 부분 리뉴얼, 전체 리뉴얼
- 공정 유형: 촬영+편집/검수, 편집/검수만 (기존 촬영본 활용)

- **[PR-005]** 간트 차트 날짜 범위는 프로젝트 생성일(또는 사용자 지정 시작일)부터 강의 지급일 +7일까지 표시
  - 배경: 프로젝트 전체 라이프사이클을 한눈에 보되, 지급일 이후 여유 기간도 확보
  - 적용 화면: 프로젝트 상세 > 일정 탭 > 장 펼침 시 미니 간트
  - 범위 밖 태스크가 있으면 자동으로 범위를 확장하며, 화살표로 오늘/일정 위치로 이동 가능

- **[PR-006]** 장(챕터) 추가와 삭제가 모두 가능하며, 삭제 시 해당 장의 모든 공정이 함께 삭제됨
  - 배경: 기획 변경으로 장 수가 줄어들 수 있음
  - 적용 화면: 프로젝트 상세 > 일정 탭 > 장 펼침 하단 삭제 버튼
  - 사전 준비(0장)는 삭제 불가

- **[PR-007]** 강(講) 단위로 공정별 결과물 파일이 연결되며, 파일 업로드 = 업무 완료
  - 배경: 외부 서비스에 파일을 올리면 자동으로 업무 완수 여부를 팔로업
  - 공정별 결과물 매핑:
    - 교안 → 교안 문서 링크 (Google Docs)
    - 촬영 → 촬영 원본 영상 링크 (Google Drive)
    - 편집 → 편집 완료 영상 링크 (Google Drive)
    - 자막 → 자막 파일 링크 (Google Drive, .srt)
    - 검수 → 백오피스 검수 페이지 링크
  - 적용 화면: 프로젝트 상세 > 작업 현황 탭 > 강별 결과물 링크

## 미결정 사항

- [ ] 챕터 간 병렬 작업이 가능한가? (예: CH1 편집 중에 CH2 촬영)
- [ ] "편집/검수만" 유형일 때 프로세스에서 생략되는 단계는?
- [ ] 리뉴얼 프로젝트에서 기존 챕터 재활용 시 어떻게 관리하는가?
- [ ] 환경 세팅(슬랙 채널, 구글 드라이브)은 시스템에서 자동화하는가, 수동인가?

- 제작 요청 폼
