# 역할 및 권한

> 시스템 사용자별 역할과 접근 가능한 기능을 정의한다.

## 이해관계자

### 규칙

- **[RL-001]** 시스템 사용자는 6가지 역할로 구분:
  - PM (Tutor Relations PM): 전체 제작 프로세스 총괄
  - 커리큘럼기획 매니저: 목차 설계, 분량 확정, 백오피스 상태 관리
  - Tutor Relations Manager: 튜터 확정 후 정보 공유
  - 튜터: 강의 촬영 (리허설 + 본촬영)
  - 편집자: 촬영 영상 편집
  - 검수자: 편집 영상 품질 검수
  - 적용 화면: 전체 (역할별 뷰 분기 시)

- **[RL-002]** PM이 프로젝트의 주 관리자이며 모든 상태 변경 권한 보유
  - 배경: 현재 MVP는 PM 대시보드 관점으로 설계
  - 적용 화면: 모든 페이지

- **[RL-003]** 영상 검수는 PM과 커리큘럼기획 매니저가 각각 독립적으로 수행
  - 배경: PM은 영상 퀄리티(음성, 화면), 커기매는 콘텐츠 정합성 관점
  - 적용 화면: 프로젝트 상세 > 영상 피드백 탭

## 미결정 사항

- [ ] 편집자/검수자/튜터가 직접 시스템에 로그인하여 상태를 변경할 수 있는가?
- [ ] PM 외에 프로젝트 상태를 변경할 수 있는 역할이 있는가?
- [ ] 한 프로젝트에 PM이 여러 명 배정될 수 있는가?

## 관련 화면

- 프로젝트 상세 전체
- 영상 피드백 탭 (`components/detail/video-feedback-*`)
