# 비즈니스 정책 인덱스

> 이 디렉토리는 실무자와의 대화에서 추출된 비즈니스 규칙을 체계적으로 관리한다.
> **백엔드 개발자가 API/DB를 설계할 때 이 문서들이 SSOT(공식 계약서) 역할**을 한다.

## 📘 가장 중요한 문서 (백엔드 개발 시 먼저 읽을 것)

| 파일 | 역할 |
|------|------|
| [정책-초안.md](정책-초안.md) | **메인 정책 문서**. 12개 섹션에 모든 비즈니스 룰이 정리됨 |
| [API-스키마.md](API-스키마.md) | **API 계약서**. 엔티티 필드 + 엔드포인트 Request/Response 명세 |
| [백엔드-할일.md](백엔드-할일.md) | 프론트에 mock으로만 되어 있고 백엔드가 구현해야 할 항목 체크리스트 |

## 📐 정책 ID 체계

- `ST-` 상태 전이 (State Transition)
- `RL-` 역할/권한 (Role)
- `PR-` 프로세스 (Process)
- `RV-` 검수/품질 (Review)
- `SC-` 일정/계산 (Schedule/Calc)
- `DT-` 데이터 규칙 (Data)
- `BZ-` 기타 비즈니스 규칙 (Business)
