# 비즈니스 정책 인덱스

> 이 디렉토리는 실무자와의 대화에서 추출된 비즈니스 규칙을 체계적으로 관리한다.
> 나중에 백엔드 개발자가 API를 설계할 때 이 문서들이 핵심 참고 자료가 된다.

## 정책 ID 체계

- `ST-` 상태 전이 (State Transition)
- `RL-` 역할/권한 (Role)
- `PR-` 프로세스 (Process)
- `RV-` 검수/품질 (Review)
- `SC-` 일정/계산 (Schedule/Calc)
- `DT-` 데이터 규칙 (Data)
- `BZ-` 기타 비즈니스 규칙 (Business)

## 문서 관리 규칙

- 새 규칙 발견 시: 해당 카테고리 파일에 추가 + 이 인덱스 업데이트
- 기존 규칙 변경 시: 변경 이력을 기록
- 미결정 사항은 `## 미결정 사항` 섹션에 체크박스로 관리
