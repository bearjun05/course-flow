## 🟡 참고 (백엔드 개발 시점에 구체화)

현재 블로커 없음. 실제 백엔드 개발에 착수할 때 아래 항목들을 구체화.

**Drive 폴더 트리 및 파일명 감지 상세** (→ `백엔드-할일.md` Drive 섹션)
- 구체적인 감시 방식 (Drive Push Notification vs 주기 폴링)
- 중복 업로드 시 처리
- 파일명 포맷 위반 시 PM에게 경고 알림을 줄지
- 자동 감지 주기/지연 허용 범위

**담당자 배정 데이터 모델** (→ `백엔드-할일.md` 데이터 모델 섹션)
- 현재 프론트: string (복수 선택 시 쉼표 구분)으로 임시 저장
- 백엔드: 배열 타입 또는 별도 `ProjectAssignment` 테이블

**슬랙 API 연동 범위**
- 실제 워크스페이스 채널 검색/멤버 조회 (Conversations API)
- Slack App 인증 방식 (봇 토큰 vs 사용자 토큰)

**강의명 변경 시 동기화**
- Drive 폴더명 변경 (folders.update)
- 슬랙 채널명 변경 (conversations.rename)
- 백오피스 API 연동
