# 상태 전이 규칙

> 프로젝트와 태스크의 상태가 어떤 조건에서 변경되는지 정의한다.

## 프로젝트 상태

### 규칙

- **[ST-001]** 프로젝트 상태는 기획 → 교안작성 → 촬영/제작 → 리뷰 → 완료 순서로 진행
  - 배경: docs/상태값목록.md 기반
  - 적용 화면: 칸반 보드, 프로젝트 상세

- **[ST-002]** "중단" 상태는 어느 단계에서든 전환 가능 (내부 사정에 의한 일시 중지/취소)
  - 적용 화면: 프로젝트 상세 > 상태 변경

- **[ST-003]** 칸반 컬럼과 프로젝트 상태의 매핑:
  - 교안작성 컬럼 ← 기획, 교안작성
  - 리허설 컬럼 ← 리허설
  - 제작 컬럼 ← 촬영/제작, 편집/검수
  - 롤아웃 컬럼 ← 리뷰, 완료
  - 적용 화면: 대시보드 칸반 보드

## 태스크 상태

### 규칙

- **[ST-004]** 공통 태스크 (프로젝트 전반 1회성): 대기 → 진행 → 완료
  - 배경: 리허설 촬영, 최종 롤아웃 등 챕터에 귀속되지 않는 과업
  - 적용 화면: 프로젝트 상세 > 일정 탭

- **[ST-005]** 챕터별 반복 태스크: 대기 → 진행 → 리뷰 → 완료
  - 배경: 교안제작-촬영-편집-검수 등 챕터마다 반복되는 공정
  - 적용 화면: 프로젝트 상세 > 일정 탭

- **[ST-006]** 챕터별 태스크는 "리뷰" 단계가 존재 (PM 확인 단계)
  - 배경: 공통 태스크와 달리 PM이 중간에 확인하는 절차가 필요
  - 적용 화면: 프로젝트 상세 > 일정 탭

## 미결정 사항

- [ ] 프로젝트 상태를 역방향으로 되돌릴 수 있는가? (예: 촬영/제작 → 교안작성)
- [ ] "중단" 상태에서 재개 시 어떤 상태로 복귀하는가?
- [ ] 태스크 "리뷰"에서 반려 시 "진행"으로 돌아가는가, "대기"로 돌아가는가?
- [ ] 모든 챕터 태스크가 완료되어야 프로젝트 상태가 자동 전이되는가?
