## 1. 강의 제작 프로세스 (단계별 전이 조건)

### ✅ 확정: 7단계 파이프라인 (기획 포함)

```
기획 → 교안 → 촬영 → 편집·자막 → 검수 → 승인 → 완료
```

| 단계 | 시작 조건 (트리거) | 종료 조건 | 담당자 |
|------|-------------------|-----------|--------|
| 기획 | 제작 요청 폼 제출 시 시작 | **기획 완료 모달에서 커리큘럼 링크 + 장/강 목록 입력 후 완료 클릭** | 커기매+PM |
| 교안 | 기획 완료 후 모든 장이 교안 단계로 시작 | 커기매가 노션 링크를 교안링크로 첨부 | 커기매 |
| 촬영 | 교안링크 첨부 완료 | 튜터가 구글 드라이브 원본영상 폴더에 파일 업로드 | 튜터 |
| 편집·자막 | 원본영상 업로드 감지 | 최종영상+최종자막 폴더에 파일 업로드 | 편집자 + 자막자 |
| 검수 | 최종파일 업로드 감지 | 검수자가 "검수완료" 버튼 클릭 | 검수자 |
| 승인 | 검수완료 클릭 | PM이 "승인" 버튼 클릭 | PM |
| 완료 | PM 승인 완료 | - (최종 단계) | - |

### ✅ 확정: 프로젝트(강의) 완료 조건

**모든 장이 "완료" 단계** 에 도달해야 프로젝트 완료 가능.

### ✅ 확정: 장 간 병렬 작업 가능

CH1 편집 중 CH2 촬영 동시 가능. 장마다 독립적 파이프라인.

### ✅ 확정: 자동 전이 vs 수동 전이 (⚠️ 백엔드 연결 필요)

| 전이 | 방식 | 트리거 |
|------|------|--------|
| 교안→촬영 | 자동 | 교안링크 등록 감지 |
| 촬영→편집·자막 | 자동 | 원본영상 폴더 파일 업로드 감지 (강별 파일명 매핑) |
| 편집·자막→검수 | 자동 | 최종영상 + 최종자막 업로드 감지 |
| 검수→승인 | **수동** | 검수자가 "검수완료" 클릭 |
| 승인→완료 | **수동** | PM이 "승인" 클릭 |

- 자동 전이 시 다음 담당자에게 슬랙 알림 (메시지 템플릿 있음 — 아래 [슬랙 알림 메시지 템플릿] 참고)
- 자동 전이된 단계도 **PM만** 강의 상세 일정 탭에서 체크 토글로 해제 가능
- 되돌리면 **해당 단계가 다시 "진행중"으로 돌아감**. 이후 단계가 있었다면 모두 연쇄적으로 해제되어 마찬가지로 "진행중" 상태로 되돌아감
- 되돌린 단계의 트리거는 재작동 (예: 슬랙 메시지 재발송)
- 되돌리기 시 **업로드된 파일은 삭제하지 않고 유지** (파일 재사용 가능)

### ✅ 확정: 상태 역방향 전환 (Q-01)

- **PM 전용** 기능. 강의 상세 페이지 **일정 탭**에서 단계 칩 체크/해제로 조작
- PM이 이전 단계로 체크 해제 → 이후 단계 자동 해제 + 해당 단계 트리거 재작동
- PM이 이후 단계로 체크 → 해당 단계 강제 시작 (정상 순서 아니어도 가능)
- 외부 관계자는 불가 (Q-04 참고)

### ✅ 확정: "중단" 재개 복구 (Q-02)

중단 직전 상태를 DB에 저장해두고, 재개 시 **자동으로 복구**.
- 프로젝트 엔티티에 "중단 직전 상태" 필드 필요
- PM이 별도로 상태를 고르는 과정 없음

### ✅ 확정: 태스크 순서 강제 (Q-04)

**기본**: 자동 전이 체인에 의해 순서가 강제됨. 이전 단계가 완료되지 않으면 다음 단계는 자동 시작되지 않음.

**예외**: PM은 일정 탭에서 체크 토글로 **임의 단계 강제 시작/해제** 가능. 외부 관계자는 불가.

**⚠️ 프론트 구현 상태**: 현재 수동 상태 변경만 지원. 자동 전이 / PM 체크 토글 / 되돌리기 연쇄 해제는 백엔드 연결 후 구현 필요.

### ✅ 확정: 프로젝트 완료 조건 (Q-05)

**완료 전환 시점**: 모든 장의 모든 강이 승인 완료되면 자동으로 프로젝트 상태 "완료" 전환.
- D-Day 당일(D-Day=0)에 완료 가능
- 롤아웃일 이전에도 완료 가능 (빨리 끝나면 일찍 내려감)

**대시보드 "진행 중" 리스트에서 내려가는 조건**:
- 프로젝트 상태가 "완료"가 되어야만 내려감
- D-Day가 지나도 아직 승인 완료 안 됐으면 리스트에 계속 남아있음 (지연 상태로 표시)

**⚠️ 프론트 구현 상태**: 자동 완료 로직 미구현. 현재는 info-guide-tab의 상태 드롭다운으로 PM이 수동 변경. 백엔드 연결 후 "모든 장·강 승인 완료 → 자동 완료" 이벤트 트리거 필요.

### ✅ 확정: 편집·자막 완료 조건

최종영상 AND 최종자막이 **모두** 업로드되어야 편집·자막 단계 완료.

### ✅ 확정: 편집·자막 데이터 모델

- UI와 진척표에서는 **한 단계**로 합쳐 표시 ("편집·자막")
- 내부 데이터(TaskType)는 **"편집"과 "자막" 두 태스크로 분리** 저장 (`/lib/types.ts`)
- 작업현황 탭에서 두 태스크가 모두 "완료"여야 합친 단계가 "완료"로 표시됨 (`/components/detail/work-status-tab.tsx`)

### ✅ 확정: 영상 검수 기능 제거 (Q-12)

시스템에서 영상 검수 기능을 **운영하지 않음**. 관련 타입/데이터/UI 모두 제거.
- `FeedbackVerdict`, `ReviewScore`, `VideoReview`, `VideoFeedback`, `PM_QUESTIONS`, `CM_QUESTIONS` 타입 제거
- `Project.videoFeedbacks` 필드 제거
- 검수자의 판단은 **"검수완료" 체크 토글**로 단순화 (현재 상태 유지)
- 영상 문제 시: **체크하지 않고 슬랙에서 직접 커뮤니케이션** (별도 시스템 기능 없음)

루트: **[팀스파르타_교육운영실_온라인강의](https://drive.google.com/drive/u/0/folders/0APsFmfSdUCjTUk9PVA)**

- 이 공유 드라이브 아래에 강의별 폴더가 자동 생성됨
- 서비스 계정 권한 부여 및 API 호출 방식은 **백엔드 구현 시 상세화 예정**

### ✅ 확정: 폴더 트리 및 파일명 포맷 (Q-06)

```
[팀스파르타_교육운영실_온라인강의]  (루트)
└── {강의명}  ← 백오피스와 동일한 이름
    └── {강의명}_v{버전}
        └── {N장_제목}  ← 예: "1장_오류 처리" (기획 완료 시 자동 생성)
            ├── 01. 원본영상
            │   └── N-N_(m)_요청사항
            │       - N-N: 장-강 (1-1, 3-2 등)
            │       - (m): 분할 촬영 합본 순서 (예: (1), (2))
            │       - 요청사항: 편집 요청사항 (예: "7분까지 사용해주세요")
            ├── 02. 편집 (vrew)
            │   └── N-N_제목
            ├── 03. 최종영상
            │   └── N-N_제목
            ├── 04. 최종자막
            │   └── N-N_제목
            └── 05. 요약노트
                └── N-N_제목
```

### ✅ 확정: 장 폴더 자동 생성 시점

**기획 완료 모달에서 장 목록을 입력하고 "완료" 클릭 시 장 폴더가 자동 생성된다.**
- 포맷: `{장번호}장_{장제목}` (예: `1장_오류 처리`)
- 이후 상세 페이지에서 장 추가 시에도 자동 폴더 생성
- 작업현황 탭의 장 헤더 "드라이브" 버튼 클릭 시 해당 장 폴더로 이동

**⚠️ 프론트 구현 상태**: 현재 `chapterDriveLinks` 배열에 **mock URL만 생성** (`projects/[id]/page.tsx` L309-314). 실제 Google Drive API 연동은 백엔드 작업 필요 (`백엔드-할일.md` 참고).

### ✅ 확정: 프로젝트 삭제 시 Drive 처리 (Q-18)

- Drive 폴더는 **삭제하지 않음**
- 폴더 이름 뒤에 `(deprecated)` 추가. 예: `기존강의명 (deprecated)`

### ✅ 확정: 자동 감지 조건 (Q-09)

**폴더 위치 + 파일명 포맷을 모두 만족해야** 자동 감지됨.
- 올바른 폴더(예: `01. 원본영상`, `03. 최종영상`, `04. 최종자막` 등)에 업로드 + 파일명이 정확한 포맷이어야 감지
- 파일명 포맷을 지키지 않은 파일은 **무시**됨 (자동 전이 트리거되지 않음)
- 백엔드에서 Drive 파일 감지 시 정규식 등으로 파일명 검증 필요

### ✅ 확정: 리뉴얼 제작 요청 시 Drive 폴더 처리

**신규 제작:**
- `{루트}/{강의명}/{강의명}_v1/` 신규 생성

**리뉴얼:**
- 제작 요청 폼에서 기존 강의 검색 (외부 API로 강의 데이터 로드)
- Drive에서 기존 강의명 폴더 존재 여부 확인
  - **폴더 있음**: 그 아래에 `{강의명}_v{다음버전}/`만 추가 생성
  - **폴더 없음**: 강의명 폴더부터 새로 생성 + 버전 폴더 생성
- 이후 하위 장 폴더는 기획 완료 모달에서 자동 생성 (동일)

## 3. 프로젝트 상태 (7가지)

### ✅ 확정

| 상태 | 의미 | 대시보드 표시 |
|------|------|--------------|
| 기획 | 초기 기획. 챕터/강 0개 가능 (기획 완료 모달에서 생성) | "진행 중" 탭 + 진척표 "기획" 열 |
| 교안 | 교안 작성 중 | 표시 |
| 촬영 | 촬영 중 | 표시 |
| 편집·검수 | 편집/자막/검수/승인 | 표시 |
| 완료 | 제작 완료 | "전체" 탭에만 표시 |
| 중단 | 일시 중지/취소 | "전체" 탭에만 표시 |

### ✅ 확정: "롤아웃" 상태 제거 (Q-16)

ProjectStatus enum에서 **"롤아웃" 제거**. 총 6개 상태로 정리됨.
- 이유: "모든 장·강 승인 완료 → 자동 완료" 규칙이 확정되어 롤아웃 과도기 상태 불필요
- 기존 mock 데이터에서 "롤아웃" 상태인 프로젝트는 적절한 상태로 마이그레이션 (예: "편집·검수" 또는 "완료")

### ✅ 확정: 대시보드 탭별 필터

- **"진행 중"**: 완료/중단 제외 (기획~롤아웃 포함)
- **"전체"**: 모두 표시 (중단 포함)
- **진척표**: 기획 포함, 완료/중단 제외 (기획은 프로젝트 단위 노드 1개로 표시)
- **요약 메트릭(지연)**: 완료 제외

## 4. 대시보드 (에듀옵스 `/`)

### ✅ 확정: 요약 메트릭

| 카드 | 값 | 설명 |
|------|-----|------|
| 오늘 태스크 | 현재 "진행" 상태인 전체 태스크 수 | **활성 프로젝트**(기획~롤아웃, 완료/중단 제외) 기준, D-Day 가까운 순 |
| 지연 | 마감 초과 프로젝트 수 | D-Day < 0 이고 완료가 아닌 프로젝트. 0일 때 초록 배경 + "없음" 표시 |

> **활성 프로젝트 정의**: `isProjectActive()` = 완료/중단이 아닌 모든 프로젝트 (기획 포함). (`/lib/utils.ts` L56-58)

### ✅ 확정: 진척표 (도트 매트릭스)

- **7단계 컬럼**: 기획 / 교안 / 촬영 / 편집·자막 / 검수 / 승인 / 완료
- **기획 열**: 프로젝트 단위 노드 1개 (장 단위 아님). 기획 상태일 때만 표시
- **도트 = 장(章) 단위**: 각 도트는 장 번호만 표시 (1, 2, 3...). 강(講) 단위 아님
- **도트 색상**: 연두 계열 가장 진한 색(#8AAE50) 일괄 통일
- **지연 도트**: 빨강(#F9919E)
- **5장 이상 연속 같은 단계**: 묶음 도트로 표시 (예: "1~5" = 1장~5장, 강 라벨 "1-1"과 혼동 방지 위해 `~` 사용)
- **행 클릭 또는 도트 클릭**: 강의 상세 페이지로 이동

### ✅ 확정: 프로젝트 단계 분류 (진척표/필터용)

프로젝트의 "현재 단계"는 `project.status`뿐 아니라 **장별 태스크 진행 상황**까지 봐서 판정한다.

- 프로젝트 상태가 `"촬영"`이더라도, **어느 장이든 편집 태스크가 `대기` 외 상태(진행/리뷰/완료)이면 "편집·검수" 단계**로 분류
- 구현: `lib/process-helpers.ts` `getEffectiveKanbanColumn()`
- 배경: 촬영과 편집이 장별로 순차 진행되므로, 편집이 시작되는 순간 프로젝트 전체 퍼널이 편집·검수 단계로 이동해야 함

### ✅ 확정: 프로젝트 카드 파이프라인 (4단계)

4단계 초록 그라데이션: 교안 / 촬영 / 편집·자막 / 검수

### ✅ 확정: 프로젝트 그리드

- **필터 기본값**: "전체 상태", "전체 사업부", "전체 트랙" (placeholder로 표시)
- **사업부**: KDT / KDC / 기타
- **숨기기 기능**: 제거됨

## 5. 역할

### ✅ 확정: 6가지 역할

| 역할 | 소속 | 주요 업무 |
|------|------|-----------|
| PM | 내부 | 전체 제작 관리, 승인 최종 확인 |

## 6. D-Day와 신호등

### ✅ 확정

- D-Day = 롤아웃일 - 오늘 (달력일, 주말 포함)
- **시간대 기준**: 한국 시간대(Asia/Seoul) **자정 기준**으로 계산
  - 예: 롤아웃일 `2026-05-01`이고 오늘이 한국 시간 `2026-04-28 23:59`이면 D-Day = 3
  - 백엔드는 UTC로 저장하더라도 D-Day 계산 시 KST로 변환 후 날짜만 비교
  - 프론트도 `new Date()`가 브라우저 로컬 시간을 반환하므로 한국 사용자 기준으로 정상 동작
- D-Day 색상 규칙 (코드: `lib/utils.ts` `getDdayColor`):
  - D-Day < 0 (마감 초과): 빨강
  - 0 ≤ D-Day ≤ 3 (**D-Day 당일 및 3일 이내 포함**): 노랑
  - 그 외: 회색
- **신호등은 전부 PM 수동 조작**. D-Day나 상태에 따른 자동 변경 없음.
  - 마감 초과되더라도 신호등은 자동으로 빨강으로 바뀌지 않음
  - PM이 프로젝트 상세 헤더에서 클릭으로만 변경

### 적용 위치

위 D-Day 색상 규칙은 D-Day가 숫자로 표시되는 모든 UI에 동일하게 적용:
- 프로젝트 상세 헤더의 D-Day 배지
- 대시보드 "진행 중" 탭의 프로젝트 카드
- 대시보드 진척표의 D-Day 칼럼
- 에듀웍스 목록의 D-Day 칼럼

- hasCurriculum (있음이면 curriculumLink, 없음이면 conceptDescription은 선택)
- needSchedule (직접 산정이면 3개 일정 날짜 필수)

### ✅ 확정: 장별 러닝타임 / 장 목록 / 강 목록

폼에서는 입력 안 함. **프로젝트 상세 페이지의 기획 완료 모달**에서 입력.
- 상세 페이지의 기본정보 패널에서 분량만 수정 가능
- 상세 페이지의 작업현황/일정에서 장 추가 가능

## 9. 에듀웍스 (`/eduworks`, `/eduworks/[id]`)

### ✅ 확정: 접근 대상

외부 관계자 4종만: **튜터, 편집자, 자막자, 검수자**
- 커기매는 에듀옵스(`/`)를 사용 → 에듀웍스 사용자 목록에서 제외
- PM도 에듀옵스 사용

### ✅ 확정: 로그인 기반 자동 선택

**프로덕션**: 로그인 정보로 현재 사용자가 자동 결정됨 → 해당 사용자의 대시보드만 표시.
**현재 프론트**: `/lib/mock-auth.ts`의 `MOCK_CURRENT_USER`로 mock 로그인 사용자 지정 (기본: 강태경, 편집자).
- 상단에 디버그용 "사용자 전환" 드롭다운 — 프로덕션 배포 시 제거
- 이름 직접 선택 방식은 폐기됨

### ✅ 확정: 에듀웍스 홈 (`/eduworks`)

**헤더:**
- `AppHeader` title="에듀웍스", "+ 강의 추가" 버튼 숨김 (외부 관계자는 프로젝트 생성 불가)

**현재 로그인 사용자 카드:**
- "안녕하세요, {이름} ({역할})" 표시
- "담당 강의 N개" 표시
- (디버그) 사용자 전환 드롭다운

**본문 (담당 강의 있음/없음):**
- 담당 0개: "담당한 강의가 없습니다"
- 담당 있음: SummaryMetrics → ProgressTable → TaskCalendar

**역할 → 담당 프로젝트 매핑:**
- 튜터: `project.tutor === 내이름`
- 편집자: `project.editor === 내이름`
- 자막자: `project.subtitleEditor === 내이름`
- 검수자: `project.reviewer === 내이름`

**역할 → 담당 단계 강조 (진척표):**
- 튜터 → "촬영" 컬럼 강조
- 편집자/자막자 → "편집·자막" 컬럼 강조
- 검수자 → "검수" 컬럼 강조
- 강조 방식: 헤더 배경 + 타임라인 영역 배경 연한 올리브색(`#EDF2DC`)

### ✅ 확정: 요약 메트릭 (에듀웍스)

- **오늘 태스크**: 내가 담당한 프로젝트에서 "진행" 상태 태스크 수
- **지연**: 내가 담당한 프로젝트 중 마감 초과 프로젝트 수

### ✅ 확정: 태스크 캘린더 (에듀웍스)

`selectedPerson` 필터 적용. 내가 담당한 태스크 타입만 표시:
- 튜터 → 교안제작, 촬영
- 편집자 → 편집
- 자막자 → 자막
- 검수자 → 검수
- `task.assignee`가 본인이면 태스크 타입 무관하게 표시

### ✅ 확정: 에듀웍스 상세 (`/eduworks/[id]`)

**탭 구성 (맨 앞부터):**
- 작업 현황 / 일정 / 이번 주

**작업 현황 탭 (`WorkStatusTab` 재사용):**
- 에듀옵스 상세의 작업 현황과 **동일한 컴포넌트** 사용
- 모든 결과물 링크는 외부 링크 열기만 가능 (링크 수정/입력 불가)
- **검수자만** 검수 토글 활성화 (`person === project.reviewer`)
- 그 외 역할은 검수 셀도 조회만
- 장 추가, 승인 토글, 교안 링크 입력 모두 비활성
- `planningComplete={true}` 고정 (블러 없음)

**기본정보 패널 (`InfoGuideTab` `readOnly` 모드):**
- 상태: 드롭다운 → 배지 (변경 불가)
- 롤아웃일 / 강의지급일: 날짜 input → 텍스트
- 장별 분량: 편집 버튼 숨김
- 메모: 읽기 전용 블록, 없으면 "메모가 없습니다"
- 바로가기: 링크 없으면 "+ 추가" 대신 "{label} 없음" 회색 표시
- 슬랙: 미등록이면 "슬랙 없음"
- 담당자: 비배정이면 "담당자 없음" (배정 버튼 숨김)
- 담당자 이름 클릭 불가 (기존 people 페이지 링크 제거됨)
- 담당자 이름 줄바꿈 방지 (truncate + whitespace-nowrap)

**헤더 (`DetailHeader` `readOnly` 모드):**
- 신호등 3개: `span` 태그로 렌더, 클릭 불가
- "⋯" 메뉴 (중단 처리/삭제): **메뉴 자체가 숨겨짐**

**일정 탭 (`MondayBoard` `readOnly` 모드):**
- 장별 6개 단계 칩 (교안/촬영/편집/자막/검수/승인): **전부 비활성**. `disabled={readOnly}` + 클릭 차단 + `cursor-default`
- 장 펼침 시 미니 간트의 일정 바 드래그 (좌/우/이동): 모두 차단
- 일정 미정 태스크를 클릭해 날짜 배정: 차단 (cursor-default, opacity-50)
- 일정 삭제 × 버튼: 숨김
- 장 추가 버튼: 숨김
- 장 삭제 버튼: 숨김

**이번 주 탭 (`WeeklyCalendar`):**
- 주간 이동 버튼: 가능
- 태스크 바 클릭 토글: **검수자만, 검수 태스크에 한해서만** 완료↔진행 전환 가능
  - 구현: 에듀웍스 페이지에서 `isReviewer`일 때만 `handleCalendarTaskToggle` 전달, 내부에서 `taskType === "검수"`인 경우에만 토글
  - 다른 역할(튜터/편집자/자막자)은 `onTaskToggle=undefined` → 클릭해도 무반응

### ✅ 확정: 자막자 업로드 UI 없음

자막자는 Google Drive 폴더에 **직접 파일을 올리면 자동 감지**. 에듀웍스 별도 업로드 UI 불필요. 작업 현황 탭에서는 결과물 링크 열기만 가능.

### ✅ 확정: 탭 기본 선택

역할 무관하게 **"작업 현황"이 기본 탭**. 사용자별 다른 기본 탭 없음.

### ✅ 확정: 외부 관계자 정보 공개 범위

민감 정보 숨김 없음. 장 수, 기획 상태, 담당자, 일정, 지급일 등 **모든 정보 표시**.

## 10. 태스크 캘린더 (`TaskCalendar` 공통 컴포넌트)

### ✅ 확정: 사용 위치

- **에듀옵스 루트(`/`)**: 사용하지 않음
- **에듀옵스 프로젝트 상세 `/projects/[id]` 작업현황 탭 하단**: 유지
- **에듀웍스 홈(`/eduworks`)**: `selectedPerson` 필터 적용

### ✅ 확정: 구성

**좌측 월간 캘린더**
- 월 이동 ← → 버튼 + "오늘" 리셋 버튼
- 주 시작: 월요일
- 오늘 셀: 검정 원 배경, 선택 셀: 베이지 배경

**날짜 셀 이벤트 바**
- 태스크 시작~마감 구간을 가로 바로 연결 (시작/중간/마감 구분 스타일)
- 색상: 완료 = 회색, 지연 = 빨강, 진행 = 장 색상
- **최대 4개까지 기본 노출**. 초과 시 "+N 더보기" 버튼 → 클릭하면 전체 펼침, "접기" 버튼으로 다시 접음
- **이벤트 바 자체는 클릭 미동작**. 셀 클릭 시 사이드 패널 선택만 변경

**우측 사이드 패널**
- 선택한 날짜의 전체 이벤트 카드 리스트
- 카드 클릭 → `/{basePath}/{projectId}?tab=work-status` 이동 (에듀웍스는 `&person=` 포함)
- 빈 날: "이 날에는 일정이 없습니다"

**이벤트 카드 내용**
- 프로젝트명 (회색)
- 장 + 공정 이름 + 색상 점
- 상태 배지: 완료 / 마감 초과 / 당일 / 시작 / 마감 / 진행 중
- 기간 (시계 아이콘 + M.d ~ M.d)
- 태스크 상태 (대기/진행/리뷰/완료)
- **담당자 이름**: `task.assignee` 우선, 없으면 역할 기반 추론 (아래 표)

### ✅ 확정: 담당자 추론 규칙 (task.assignee 없을 때)

| 태스크 타입 | 담당자 |
|-------------|--------|
| 커리큘럼 기획, 승인 | `project.curriculumManager` |
| 교안제작, 촬영 | `project.tutor` |
| 편집 | `project.editor` |
| 자막 | `project.subtitleEditor` |
| 검수 | `project.reviewer` |
| 기타 | 미표시 |

### ✅ 확정: 필터 규칙 (`selectedPerson` 있을 때)

- 튜터 → 교안제작, 촬영
- 편집자 → 편집
- 자막자 → 자막
- 검수자 → 검수
- 커기매 → 커리큘럼 기획, 승인
- `task.assignee === person` → 태스크 타입 무관
- **롤아웃 바 제외** (에듀웍스 외부 관계자에게는 롤아웃 마감 미표시)

### ✅ 확정: 지연 판정

- 이벤트 바 빨강(`#DC2626`) 조건: 태스크 완료 아니고 `endDate < 오늘`
- 카드 테두리/배경도 빨강 계열

### ✅ 확정: 리뷰 상태 별도 표시 없음

캘린더는 대기/진행/완료 중심. "리뷰" 상태도 그냥 `task.status` 문자열로만 노출 (전용 색/배지 없음).

## 11. 슬랙 자동 알림 템플릿 (Q-14)

자동 전이 또는 검수/승인 이벤트 발생 시 프로젝트 슬랙 채널에 메시지 자동 전송.
백엔드에서 Slack API로 전송. 멘션은 해당 역할 담당자를 태그.

### 교안 완료 (커기매 교안링크 등록 → 촬영 자동 전이)

```
[교안 제작 완료] @튜터 @커기매 @PM

N-N 교안 작성이 완료되었습니다.
- 교안 확인하기: {링크}
```

### 촬영 완료 (튜터 원본영상 업로드 → 편집·자막 자동 전이)

**편집 담당자 배정 전:**
```
[촬영 완료] @PM

N-N 촬영이 완료되었습니다.
- 촬영 영상 확인하기: {링크}
```

**편집 담당자 배정 후:**
```
[촬영 완료] @PM @편집담당자

N-N 촬영이 완료되었습니다.
- 촬영 영상 확인하기: {링크}

@편집담당자 님, 강의 촬영이 완료되었습니다. 편집 준비를 해주세요!
```

### 편집·자막 완료 (자막자 최종영상+최종자막 업로드 → 검수 자동 전이)

```
[편집·자막 완료] @PM

N-N 편집 및 자막 작업이 완료되었습니다.
- 백오피스 바로가기: {링크}

@자막담당자 님, 백오피스 업로드 부탁 드립니다!
```

### 검수 완료 (검수자 "검수완료" 클릭 → 승인 단계 시작)

```
[검수 완료] @PM

N-N 검수 완료되었습니다. 확인 후 승인 단계를 진행해 주세요.
- 백오피스 바로가기: {링크}
```

### 승인 완료 (PM "승인" 클릭)

```
[승인 완료] @편집 @자막 @검수 cc. @PM

N-N 최종 승인이 완료되었습니다. 수고하셨습니다!
- 백오피스 바로가기: {링크}
```

### ✅ 확정: 변수 치환 규칙

- `{링크}`: 해당 단계에 맞는 URL (교안 링크 / Drive 원본영상 폴더 / 편집 영상 / 백오피스 페이지)
- `N-N`: 장-강 라벨 (예: 1-1, 3-2)
- `@역할`: 해당 프로젝트의 역할 담당자 슬랙 멘션
  - **역할에 담당자가 복수 배정되어 있으면 전원을 @멘션한다** (예: 편집담당자 2명이면 `@편집자1 @편집자2` 모두 태그)
  - 담당자가 0명인 역할은 템플릿에서 해당 멘션 토큰 생략

## 12. 사이드바 / 네비게이션

### ⚠️ 임시 구현 (향후 제거/교체 예정)

현재 좌측 사이드바(대시보드 / 에듀웍스 메뉴)와 공통 헤더(`AppHeader`)는 **본 프론트가 독립 앱으로 동작할 때의 디버그 편의용**.

**실제 배포 시:**

따라서 현재 사이드바/헤더의 UI·메뉴 구성은 최종 정책 대상이 아니며, 통합 시점에 각 상위 시스템의 네비게이션 규약을 따른다.
