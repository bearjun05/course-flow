## 1. 강의 제작 프로세스 (단계별 전이 조건)

### ✅ 확정: 7단계 파이프라인 (기획 포함)

```
기획 → 교안 → 촬영 → 편집·자막 → 검수 → 승인 → 완료
```

| 단계 | 시작 조건 (트리거) | 종료 조건 | 담당자 |
|------|-------------------|-----------|--------|
| 기획 | 제작 요청 폼 제출 시 시작 | 사전 준비 완료 (체크리스트: 커리큘럼 링크, 롤아웃/지급일, 장별 분량) | 커기매+PM |
| 교안 | 기획 완료 후 모든 장이 교안 단계로 시작 | 커기매가 노션 링크를 교안링크로 첨부 | 커기매 |
| 촬영 | 교안링크 첨부 완료 | 튜터가 구글 드라이브 원본영상 폴더에 파일 업로드 | 튜터 |
| 편집·자막 | 원본영상 업로드 감지 | 최종영상+최종자막 폴더에 파일 업로드 | 편집자 + 자막자 |
| 검수 | 최종파일 업로드 감지 | 검수자가 "검수완료" 버튼 클릭 | 검수자 |
| 승인 | 검수완료 클릭 | PM이 "승인" 버튼 클릭 | PM |
| 완료 | PM 승인 완료 | - (최종 단계) | - |

### ✅ 확정: 프로젝트(강의) 완료 조건

**모든 장이 "완료" 단계** 에 도달해야 프로젝트 완료 가능.

### ✅ 확정: 장 간 병렬 작업 가능

CH1 편집 중 CH2 촬영 동시 가능. 장마다 독립적 파이프라인.

### ✅ 확정: 자동 전이 vs 수동 전이

| 전이 | 방식 | 트리거 |
|------|------|--------|
| 교안→촬영 | 자동 | 교안링크 등록 감지 |
| 촬영→편집·자막 | 자동 | 원본영상 폴더 파일 업로드 감지 (강별 파일명 매핑) |
| 편집·자막→검수 | 자동 | 최종영상 + 최종자막 업로드 감지 |
| 검수→승인 | **수동** | 검수자가 "검수완료" 클릭 |
| 승인→완료 | **수동** | PM이 "승인" 클릭 |

- 자동 전이 시 다음 담당자에게 슬랙 알림
- 자동 전이된 단계도 캘린더에서 클릭하면 **되돌리기 가능** (확인 alert 표시)
- 되돌리면 해당 단계 이후 모든 단계가 연쇄적으로 해제됨

### ✅ 확정: 프로젝트 완료 조건

모든 장의 모든 강이 승인 완료되면 자동으로 프로젝트 완료. (롤아웃 수동 클릭 없음)

### ✅ 확정: 편집·자막 완료 조건

최종영상 AND 최종자막이 **모두** 업로드되어야 편집·자막 단계 완료.

## 2. 구글 드라이브 폴더 구조

### ✅ 확정: 폴더 트리

```
{루트 디렉토리}  ← 환경변수로 설정
└── {강의명}  ← 백오피스와 동일한 이름
    └── {강의명}_v{버전}
        └── {N장}
            ├── 01. 원본영상       ← 튜터가 업로드
            ├── 02. 편집 (vrew)    ← 편집자가 업로드
```

## 3. 프로젝트 상태 (7가지)

### ✅ 확정

| 상태 | 의미 | 대시보드 표시 |
|------|------|--------------|
| 기획 | 초기 기획. 챕터 0개/튜터 미정 가능 | "진행 중" 탭에만 표시. 메트릭/진척표 제외 |
| 교안 | 교안 작성 중 | 표시 |
| 촬영 | 촬영 중 | 표시 |
| 편집·검수 | 편집/자막/검수/승인 | 표시 |
| 롤아웃 | 출시 준비 | 표시 |
| 완료 | 제작 완료 | "전체" 탭에만 표시 |
| 중단 | 일시 중지/취소 | "전체" 탭에만 표시 |

### ✅ 확정: 대시보드 탭별 필터

- **"진행 중"**: 완료/중단 제외 (기획~롤아웃)
- **"전체"**: 모두 표시 (중단 포함)
- **요약 메트릭/진척표**: 기획/완료/중단 제외 (교안~롤아웃만)

## 4. 대시보드 (에듀옵스 `/`)

### ✅ 확정: 요약 메트릭

| 카드 | 값 | 설명 |
|------|-----|------|
| 오늘 태스크 | 현재 "진행" 상태인 전체 태스크 수 | 활성 프로젝트(교안~롤아웃) 기준, D-Day 가까운 순 |
| 지연 | 마감 초과 프로젝트 수 | D-Day < 0 이고 완료가 아닌 프로젝트 |

### ✅ 확정: 진척표 (도트 매트릭스)

- **7단계 컬럼**: 기획 / 교안 / 촬영 / 편집·자막 / 검수 / 승인 / 완료
- **기획 열**: 프로젝트 단위 노드 1개 (장 단위 아님). 사전 준비 완료 여부만 표시
- **도트 = 장(章) 단위**: 각 도트는 장 번호만 표시 (1, 2, 3...). 강(講) 단위는 표시하지 않음
- **도트 색상**: 연두 계열 가장 진한 색(#8AAE50) 일괄 통일
- **지연 도트**: 빨강(#F9919E)
- **5장 이상 연속 같은 단계**: 묶음 도트로 표시 (예: "1-5" = 1장~5장)
- **행 클릭 또는 도트 클릭**: 강의 상세 페이지로 이동

### ✅ 확정: 프로젝트 그리드

- **필터 기본값**: "전체 상태", "전체 사업부" (식별 가능한 라벨)
- **사업부**: KDT / KDC / 기타
- **숨기기 기능**: 제거됨

## 5. 역할

### ✅ 확정: 6가지 역할

| 역할 | 소속 | 주요 업무 |
|------|------|-----------|
| PM | 내부 | 전체 제작 관리, 승인 최종 확인 |
| 커기매 | 내부 | 커리큘럼 기획, 교안링크 첨부, 콘텐츠 검수 |
| 튜터 | 외부 | 강의 촬영, 원본영상 업로드 |
| 편집자 | 외부 | 영상 편집 (vrew) |
| 자막자 | 외부 | 최종영상 + 최종자막 + 요약노트 제작 |
| 검수자 | 외부 | 영상 품질/내용 검수, 검수완료 처리 |

### ✅ 확정: 시스템 접근

- **에듀옵스** (`/`): PM + 커기매 (내부)
- **에듀웍스** (`/eduworks`): 튜터 + 편집자 + 자막자 + 검수자 (외부)

## 6. D-Day와 신호등

### ✅ 확정

- D-Day = 롤아웃일 - 오늘 (달력일, 주말 포함)
- D-Day 색상: 마감초과=빨강, 3일이내=노랑, 그 외=회색
- 신호등: 마감 초과 시 자동 빨강. 노랑은 PM 수동 설정.

## 7. 프로젝트 상세 (에듀옵스 `/projects/[id]`)

### ✅ 확정: 헤더

- 중단 처리 = 상태 변경 (복구 가능), 삭제 = 완전 제거 (복구 불가)
- 신호등: PM이 클릭으로 변경
- 버전: v2.0 이상이면 다른 버전 드롭다운 연결

### ✅ 확정: 기본정보 패널

**담당자 배정:**
- 슬랙 채널 등록 → 해당 채널 참여자 목록에서 드롭다운 검색으로 선택
- 역할별 **2명 이상 중복 선택** 가능 (UI: 가로 줄바꿈 없이 컴팩트하게)
- PM도 프로젝트마다 다를 수 있음 (고정값 아님)
- 교안링크는 **커기매가 직접 입력** (PM이 대신 입력 X)

**슬랙 연동:**
- 바로가기에서 슬랙 채널을 워크스페이스 검색+드롭다운으로 선택
- 채널 선택 완료 후 → 담당자 섹션에서 해당 채널 멤버를 드롭다운으로 표시

**바로가기 5개:**
- 교안, 드라이브, 백오피스, 커리큘럼, 슬랙
- 모두 처음에는 "+" 상태 → 클릭해서 링크 등록 → 등록 후 열기
- 드라이브는 프로젝트 생성 시 폴더 자동 생성 → 링크 자동 연결

**장별 러닝타임:**
- 제작 요청 폼에서 초기 입력 → 상세에서 수정 + **추가 가능**

### ✅ 확정: 작업현황 탭

- **기획 행**: 최상단에 "사전 준비" 행. 완료 모달(체크리스트 3개) 통과해야 완료
  - 체크: 커리큘럼 링크 등록 / 롤아웃·지급일 등록 / 장별 분량 입력
  - 기획 미완료 시 아래 장 목록이 블러 처리됨
- **장 추가 가능**: 작업현황에서 장을 추가할 수 있음 (드라이브 폴더도 함께 생성)
- **장별 분량**: 작업현황/일정에서 장 추가 시 자동 반영. 분량만 수정 가능 (분량 탭에서 장 추가 불가)
- **6열 구성**: 교안 / 촬영 / 편집·자막 / 검수 / 승인 / 완료
- **강(講) 단위 매트릭스**: 행=강(1-1, 1-2...), 열=공정
- **현재 단계 하이라이트**: 각 강의 현재 진행 단계 셀에 배경색 표시
- **완료 열**: 승인 완료 시 체크 표시, 미완료 시 빈 원
- **교안 열**: 커기매가 링크 직접 입력 (인라인)
- **촬영/편집·자막 열**: 해당 장 드라이브 폴더 열기 (파일명으로 강별 매핑)
- **검수**: 클릭 → 해당 강 검수 완료 처리
- **승인**: 클릭 → 해당 강 승인 완료 처리
- **장 헤더 드라이브 버튼**: 해당 장의 드라이브 폴더 열기

### ✅ 확정: 일정 탭 (먼데이보드)

- 태스크 일정 드래그: **누구나** 할 수 있음 (PM 전용 아님)
- 장 추가/삭제 가능
- **롤아웃 행 없음**: 모든 장 완료 시 자동으로 프로젝트 완료
- **승인 오른쪽에 완료 표시**: 승인 태스크가 완료되면 "완료" 칩 표시

### ✅ 확정: 캘린더 탭 (주간뷰)

- 태스크 클릭 → 상태 되돌리기 기능 (자동 전이된 것도 해제 가능)
- 되돌리면 **해당 단계 이후 모든 단계가 연쇄 해제**
- **확인 alert 표시**: "이 단계 시작으로 돌아갑니다. 진행하시겠습니까?"

- 프로젝트 생성 (초기 상태: "기획")

폼에 임시저장 기능 없음. 한 번에 작성해서 제출.

폼에서는 입력 안 함. **프로젝트 상세 페이지**에서만 입력/수정/추가 가능.
