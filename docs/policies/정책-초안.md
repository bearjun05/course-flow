## 1. 강의 제작 프로세스 (단계별 전이 조건)

### ✅ 확정: 7단계 파이프라인 (기획 포함)

```
기획 → 교안 → 촬영 → 편집·자막 → 검수 → 승인 → 완료
```

| 단계 | 시작 조건 (트리거) | 종료 조건 | 담당자 |
|------|-------------------|-----------|--------|
| 기획 | 제작 요청 폼 제출 시 시작 | **기획 완료 모달에서 커리큘럼 링크 + 장/강 목록 입력 후 완료 클릭** | 커기매+PM |
| 교안 | 기획 완료 후 모든 장이 교안 단계로 시작 | 커기매가 노션 링크를 교안링크로 첨부 | 커기매 |
| 촬영 | 교안링크 첨부 완료 | 튜터가 구글 드라이브 원본영상 폴더에 파일 업로드 | 튜터 |
| 편집·자막 | 원본영상 업로드 감지 | 최종영상+최종자막 폴더에 파일 업로드 | 편집자 + 자막자 |
| 검수 | 최종파일 업로드 감지 | 검수자가 "검수완료" 버튼 클릭 | 검수자 |
| 승인 | 검수완료 클릭 | PM이 "승인" 버튼 클릭 | PM |
| 완료 | PM 승인 완료 | - (최종 단계) | - |

### ✅ 확정: 프로젝트(강의) 완료 조건

**모든 장이 "완료" 단계** 에 도달해야 프로젝트 완료 가능.

### ✅ 확정: 장 간 병렬 작업 가능

CH1 편집 중 CH2 촬영 동시 가능. 장마다 독립적 파이프라인.

### ✅ 확정: 자동 전이 vs 수동 전이 (⚠️ 백엔드 연결 필요)

| 전이 | 방식 | 트리거 |
|------|------|--------|
| 교안→촬영 | 자동 | 교안링크 등록 감지 |
| 촬영→편집·자막 | 자동 | 원본영상 폴더 파일 업로드 감지 (강별 파일명 매핑) |
| 편집·자막→검수 | 자동 | 최종영상 + 최종자막 업로드 감지 |
| 검수→승인 | **수동** | 검수자가 "검수완료" 클릭 |
| 승인→완료 | **수동** | PM이 "승인" 클릭 |

- 자동 전이 시 다음 담당자에게 슬랙 알림
- 자동 전이된 단계도 캘린더에서 클릭하면 **되돌리기 가능** (확인 alert 표시)
- 되돌리면 해당 단계 이후 모든 단계가 연쇄적으로 해제됨

**⚠️ 프론트 구현 상태**: 현재 수동 상태 변경만 지원. 파일 업로드 감지 기반 자동 전이 / 되돌리기 연쇄 해제는 백엔드 연결 후 구현 필요.

### ✅ 확정: 프로젝트 완료 조건

모든 장의 모든 강이 승인 완료되면 자동으로 프로젝트 상태가 "완료"로 전환. (롤아웃 수동 클릭 없음)

**⚠️ 프론트 구현 상태**: 자동 완료 로직 미구현. 현재는 info-guide-tab의 상태 드롭다운으로 PM이 수동 변경. 백엔드 연결 후 "모든 장·강 승인 완료 → 자동 완료" 이벤트 트리거 필요.

### ✅ 확정: 편집·자막 완료 조건

최종영상 AND 최종자막이 **모두** 업로드되어야 편집·자막 단계 완료.

### ✅ 확정: 편집·자막 데이터 모델

- UI와 진척표에서는 **한 단계**로 합쳐 표시 ("편집·자막")
- 내부 데이터(TaskType)는 **"편집"과 "자막" 두 태스크로 분리** 저장 (`/lib/types.ts`)
- 작업현황 탭에서 두 태스크가 모두 "완료"여야 합친 단계가 "완료"로 표시됨 (`/components/detail/work-status-tab.tsx`)

## 2. 구글 드라이브 폴더 구조

### ✅ 확정: 폴더 트리

```
{루트 디렉토리}  ← 환경변수로 설정
└── {강의명}  ← 백오피스와 동일한 이름
    └── {강의명}_v{버전}
        └── {N장_제목}  ← 예: "1장_오류 처리" (기획 완료 시 자동 생성)
            ├── 01. 원본영상       ← 튜터가 업로드
            ├── 02. 편집 (vrew)    ← 편집자가 업로드
            ├── 03. 최종영상       ← 자막자가 업로드
            ├── 04. 최종자막       ← 자막자가 업로드
            └── 05. 요약노트       ← 자막자가 업로드 (JSON)
```

### ✅ 확정: 장 폴더 자동 생성 시점

**기획 완료 모달에서 장 목록을 입력하고 "완료" 클릭 시 장 폴더가 자동 생성된다.**
- 포맷: `{장번호}장_{장제목}` (예: `1장_오류 처리`)
- 이후 상세 페이지에서 장 추가 시에도 자동 폴더 생성
- 작업현황 탭의 장 헤더 "드라이브" 버튼 클릭 시 해당 장 폴더로 이동

**⚠️ 프론트 구현 상태**: 현재 `chapterDriveLinks` 배열에 **mock URL만 생성** (`projects/[id]/page.tsx` L309-314). 실제 Google Drive API 연동은 백엔드 작업 필요.

## 3. 프로젝트 상태 (7가지)

### ✅ 확정

| 상태 | 의미 | 대시보드 표시 |
|------|------|--------------|
| 기획 | 초기 기획. 챕터/강 0개 가능 (기획 완료 모달에서 생성) | "진행 중" 탭 + 진척표 "기획" 열 |
| 교안 | 교안 작성 중 | 표시 |
| 촬영 | 촬영 중 | 표시 |
| 편집·검수 | 편집/자막/검수/승인 | 표시 |
| 롤아웃 | 출시 준비 (승인 완료 후 완료 직전 상태) | 표시 |
| 완료 | 제작 완료 | "전체" 탭에만 표시 |
| 중단 | 일시 중지/취소 | "전체" 탭에만 표시 |

> **"롤아웃" 상태 활용**: 현재 7개 상태 enum이 모두 존재하지만, 일정 탭에서는 롤아웃 행이 제거됨. "모든 장·강 승인 완료 → 자동 '완료' 전환" 규칙이 적용되면 "롤아웃"은 PM이 수동으로 잠깐 전환하는 과도기 상태로 사용 가능. (⚠️ 백엔드 연결 후 명확한 역할 재정의 필요)

### ✅ 확정: 대시보드 탭별 필터

- **"진행 중"**: 완료/중단 제외 (기획~롤아웃 포함)
- **"전체"**: 모두 표시 (중단 포함)
- **진척표**: 기획 포함, 완료/중단 제외 (기획은 프로젝트 단위 노드 1개로 표시)
- **요약 메트릭(지연)**: 완료 제외

## 4. 대시보드 (에듀옵스 `/`)

### ✅ 확정: 요약 메트릭

| 카드 | 값 | 설명 |
|------|-----|------|
| 오늘 태스크 | 현재 "진행" 상태인 전체 태스크 수 | **활성 프로젝트**(기획~롤아웃, 완료/중단 제외) 기준, D-Day 가까운 순 |
| 지연 | 마감 초과 프로젝트 수 | D-Day < 0 이고 완료가 아닌 프로젝트. 0일 때 초록 배경 + "없음" 표시 |

> **활성 프로젝트 정의**: `isProjectActive()` = 완료/중단이 아닌 모든 프로젝트 (기획 포함). (`/lib/utils.ts` L56-58)

### ✅ 확정: 진척표 (도트 매트릭스)

- **7단계 컬럼**: 기획 / 교안 / 촬영 / 편집·자막 / 검수 / 승인 / 완료
- **기획 열**: 프로젝트 단위 노드 1개 (장 단위 아님). 기획 상태일 때만 표시
- **도트 = 장(章) 단위**: 각 도트는 장 번호만 표시 (1, 2, 3...). 강(講) 단위 아님
- **도트 색상**: 연두 계열 가장 진한 색(#8AAE50) 일괄 통일
- **지연 도트**: 빨강(#F9919E)
- **5장 이상 연속 같은 단계**: 묶음 도트로 표시 (예: "1~5" = 1장~5장, 강 라벨 "1-1"과 혼동 방지 위해 `~` 사용)
- **행 클릭 또는 도트 클릭**: 강의 상세 페이지로 이동

### ✅ 확정: 프로젝트 카드 파이프라인 (4단계)

4단계 초록 그라데이션: 교안 / 촬영 / 편집·자막 / 검수

### ✅ 확정: 프로젝트 그리드

- **필터 기본값**: "전체 상태", "전체 사업부", "전체 트랙" (placeholder로 표시)
- **사업부**: KDT / KDC / 기타

## 5. 역할

### ✅ 확정: 6가지 역할

| 역할 | 소속 | 주요 업무 |
|------|------|-----------|
| PM | 내부 | 전체 제작 관리, 승인 최종 확인 |

| 검수자 | 외부 | 영상 품질/내용 검수, 검수완료 처리 |

- **에듀옵스** (`/`): PM + 커기매 (내부)

**매트릭스 구조 (`GRID_COLS`):**
- 그리드: `[강의명 | 교안 | 촬영 | 편집·자막 | 검수 | 승인 | 완료 | 진행률(100px)]`
- 6열 공정 구성 (icon + label 헤더): 교안/촬영/편집·자막/검수/승인/완료

**장 헤더 행:**
- 좌측 3px 장 컬러 보더
- "N장 + 장 제목" 표시 + (있으면) "드라이브" 버튼
- 우측: 장별 진척률 바 + "N강" 표시

**강 행 (Lecture 단위):**
- 행당 강 라벨 (`1-1`) + 강 제목
- 셀별 동작:
  - **교안제작**: URL 있으면 외부링크 버튼, 없으면 인라인 입력 모드 진입 버튼 (커기매 입력)
  - **촬영**: URL 있으면 외부링크 열기, 없으면 점선 빈 원
  - **편집·자막(편집 키)**: URL 있으면 외부링크, 없으면 점선 빈 원
  - **검수**: 클릭 토글 (`lecture.reviewed` boolean)
  - **승인**: 클릭 토글 (`lecture.approved` boolean)
  - **완료**: 승인 완료 시 초록 체크원, 미완료 시 점선 빈 원
- 현재 단계 셀은 배경색 하이라이트(`bg-accent/20`). 판단 기준:
  - 승인 완료 → 완료
  - 검수 완료(reviewed) → 승인
  - 편집/자막 URL 둘 다 있음 → 검수
  - 촬영 URL 있음 → 편집·자막
  - 교안 링크 있음 → 촬영
  - 그 외 → 교안

**장 추가:**
- 기획 완료 후에만 활성화
- 하단 점선 버튼 "장 추가" → `onAddChapter()` → 6개 태스크 + duration=0 자동 추가

**⚠️ 미구현 항목:**
- 편집·자막 셀 = 해당 장 드라이브 폴더 열기 (현재는 lecture.editedVideoUrl만 열음)
- 장 헤더 드라이브 버튼: chapterDriveLinks 존재해야 표시, 실제 폴더 URL로 이동 (현재 mock URL)

### ✅ 확정: 일정 탭 (먼데이보드)

- 태스크 일정 드래그: **누구나** 할 수 있음 (PM 전용 아님)
- 장 추가/삭제 가능
- **롤아웃 행 없음**: 모든 장 완료 시 자동으로 프로젝트 완료
- **승인 오른쪽에 완료 표시**: 승인 태스크가 완료되면 "완료" 칩 표시

### ✅ 확정: 캘린더 탭 (주간뷰)

**현재 구현 (`weekly-calendar.tsx`):**
- 태스크 바 클릭 시 `onTaskToggle(taskId)` 호출 → status 단순 토글 (완료 ↔ 진행)
- 주간 이동(이전/다음/이번 주) 버튼
- 태스크별 일정 범위 바 + 담당자 + 업로드 진행률 표시

**⚠️ 정책(미구현):**
- 태스크 클릭 → 상태 되돌리기 기능 (자동 전이된 것도 해제)
- 되돌리면 **해당 단계 이후 모든 단계가 연쇄 해제**
- **확인 alert 표시**: "이 단계 시작으로 돌아갑니다. 진행하시겠습니까?"

## 8. 제작요청 폼 (`/projects/new`)

### ✅ 확정: 작성자

**커기매(커리큘럼기획매니저)**가 작성. PM에게 제작을 요청하는 폼.

### ✅ 확정: 질문 8개

| # | 질문 | 필수 | 조건부 |
|---|------|------|--------|
| Q1 | 강의명 | O | - |
| Q2 | 튜터 배정 여부 | O | 배정완료→이름 입력 |
| Q3 | 사용처 (KDT/KDC/기타) | O | KDT→트랙 선택, 기타→직접입력 |
| Q4 | 제작 유형 (신규/리뉴얼) | O | 리뉴얼→부분/전체 + 이전 강의명 + 범위 |
| Q5 | 예상 분량 (시간) + 챕터 수 | O(시간만) | - |
| Q6 | 커리큘럼 초안 | O | 있음→링크, 없음→컨셉 설명 |
| Q7 | 롤아웃 + 강의지급일 | O | - |
| Q8 | 세부 제작 일정 | O | 직접산정→3단계 날짜, PM요청→PM이 산정 |

### ✅ 확정: 제출 시 동작

- 프로젝트 생성 (초기 상태: **"기획"**)
- 구글 드라이브 루트 폴더 자동 생성 (`{강의명}/{강의명}_v{버전}/`)
- 장 폴더는 **이후 기획 완료 모달에서 생성** (폼 제출 시점 아님)
- 완료 화면 → "프로젝트 페이지로 이동" 버튼

**⚠️ 프론트 구현 상태**: 실제 백엔드 없이 `proj-${Date.now()}` ID만 생성하고 상세 페이지로 이동. DB 저장, Drive API 호출, 슬랙 알림은 백엔드 연결 후 구현.

### ✅ 확정: 임시저장 없음

폼에 임시저장 기능 없음. 한 번에 작성해서 제출.

### ✅ 확정: 제출 비활성화 조건

필수 항목 누락 시 제출 버튼 disabled. `isComplete` 조건:
- title, tutorAssigned, businessUnit (KDT면 trackName, 기타면 businessUnitOther)
- productionType (리뉴얼이면 renewalType)
- rolloutDate, paymentDate, estimatedDuration
- hasCurriculum (있음이면 curriculumLink, 없음이면 conceptDescription은 선택)
- needSchedule (직접 산정이면 3개 일정 날짜 필수)

### ✅ 확정: 장별 러닝타임 / 장 목록 / 강 목록

폼에서는 입력 안 함. **프로젝트 상세 페이지의 기획 완료 모달**에서 입력.
- 상세 페이지의 기본정보 패널에서 분량만 수정 가능
- 상세 페이지의 작업현황/일정에서 장 추가 가능

## 9. 에듀웍스 (`/eduworks`, `/eduworks/[id]`)

### ✅ 확정: 접근 대상

외부 관계자 4종만: **튜터, 편집자, 자막자, 검수자**
- 커기매는 에듀옵스(`/`)를 사용 → 에듀웍스 사용자 목록에서 제외
- PM도 에듀옵스 사용

### ✅ 확정: 로그인 기반 자동 선택

**프로덕션**: 로그인 정보로 현재 사용자가 자동 결정됨 → 해당 사용자의 대시보드만 표시.
**현재 프론트**: `/lib/mock-auth.ts`의 `MOCK_CURRENT_USER`로 mock 로그인 사용자 지정 (기본: 강태경, 편집자).
- 상단에 디버그용 "사용자 전환" 드롭다운 — 프로덕션 배포 시 제거
- 이름 직접 선택 방식은 폐기됨

### ✅ 확정: 에듀웍스 홈 (`/eduworks`)

**헤더:**
- `AppHeader` title="에듀웍스", "+ 강의 추가" 버튼 숨김 (외부 관계자는 프로젝트 생성 불가)

**현재 로그인 사용자 카드:**
- "안녕하세요, {이름} ({역할})" 표시
- "담당 강의 N개" 표시
- (디버그) 사용자 전환 드롭다운

**본문 (담당 강의 있음/없음):**
- 담당 0개: "담당한 강의가 없습니다"
- 담당 있음: SummaryMetrics → ProgressTable → TaskCalendar

**역할 → 담당 프로젝트 매핑:**
- 튜터: `project.tutor === 내이름`
- 편집자: `project.editor === 내이름`
- 자막자: `project.subtitleEditor === 내이름`
- 검수자: `project.reviewer === 내이름`

**역할 → 담당 단계 강조 (진척표):**
- 튜터 → "촬영" 컬럼 강조
- 편집자/자막자 → "편집·자막" 컬럼 강조
- 검수자 → "검수" 컬럼 강조
- 강조 방식: 헤더 배경 + 타임라인 영역 배경 연한 올리브색(`#EDF2DC`)

### ✅ 확정: 요약 메트릭 (에듀웍스)

- **오늘 태스크**: 내가 담당한 프로젝트에서 "진행" 상태 태스크 수
- **지연**: 내가 담당한 프로젝트 중 마감 초과 프로젝트 수

### ✅ 확정: 태스크 캘린더 (에듀웍스)

`selectedPerson` 필터 적용. 내가 담당한 태스크 타입만 표시:
- 튜터 → 교안제작, 촬영
- 편집자 → 편집
- 자막자 → 자막
- 검수자 → 검수
- `task.assignee`가 본인이면 태스크 타입 무관하게 표시

### ✅ 확정: 에듀웍스 상세 (`/eduworks/[id]`)

**탭 구성 (맨 앞부터):**
- 작업 현황 / 일정 / 이번 주

**작업 현황 탭 (`WorkStatusTab` 재사용):**
- 에듀옵스 상세의 작업 현황과 **동일한 컴포넌트** 사용
- 모든 결과물 링크는 외부 링크 열기만 가능 (링크 수정/입력 불가)
- **검수자만** 검수 토글 활성화 (`person === project.reviewer`)
- 그 외 역할은 검수 셀도 조회만
- 장 추가, 승인 토글, 교안 링크 입력 모두 비활성
- `planningComplete={true}` 고정 (블러 없음)

**기본정보 패널 (`InfoGuideTab` `readOnly` 모드):**
- 상태: 드롭다운 → 배지 (변경 불가)
- 롤아웃일 / 강의지급일: 날짜 input → 텍스트
- 장별 분량: 편집 버튼 숨김
- 메모: 읽기 전용 블록, 없으면 "메모가 없습니다"
- 바로가기: 링크 없으면 "+ 추가" 대신 "{label} 없음" 회색 표시
- 슬랙: 미등록이면 "슬랙 없음"
- 담당자: 비배정이면 "담당자 없음" (배정 버튼 숨김)
- 담당자 이름 클릭 불가 (기존 people 페이지 링크 제거됨)
- 담당자 이름 줄바꿈 방지 (truncate + whitespace-nowrap)

**헤더 (`DetailHeader` `readOnly` 모드):**
- 신호등 3개: `span` 태그로 렌더, 클릭 불가
- "⋯" 메뉴 (중단 처리/삭제): **메뉴 자체가 숨겨짐**

**일정 탭 (`MondayBoard` `readOnly` 모드):**
- 장별 6개 단계 칩 (교안/촬영/편집/자막/검수/승인): **전부 비활성**. `disabled={readOnly}` + 클릭 차단 + `cursor-default`
- 장 펼침 시 미니 간트의 일정 바 드래그 (좌/우/이동): 모두 차단
- 일정 미정 태스크를 클릭해 날짜 배정: 차단 (cursor-default, opacity-50)
- 일정 삭제 × 버튼: 숨김
- 장 추가 버튼: 숨김
- 장 삭제 버튼: 숨김

**이번 주 탭 (`WeeklyCalendar`):**
- 주간 이동 버튼: 가능
- 태스크 바 클릭 토글: **검수자만, 검수 태스크에 한해서만** 완료↔진행 전환 가능
  - 구현: 에듀웍스 페이지에서 `isReviewer`일 때만 `handleCalendarTaskToggle` 전달, 내부에서 `taskType === "검수"`인 경우에만 토글
  - 다른 역할(튜터/편집자/자막자)은 `onTaskToggle=undefined` → 클릭해도 무반응

### ✅ 확정: 자막자 업로드 UI 없음

자막자는 Google Drive 폴더에 **직접 파일을 올리면 자동 감지**. 에듀웍스 별도 업로드 UI 불필요. 작업 현황 탭에서는 결과물 링크 열기만 가능.

### ✅ 확정: 탭 기본 선택

역할 무관하게 **"작업 현황"이 기본 탭**. 사용자별 다른 기본 탭 없음.

### ✅ 확정: 외부 관계자 정보 공개 범위

민감 정보 숨김 없음. 장 수, 기획 상태, 담당자, 일정, 지급일 등 **모든 정보 표시**.
