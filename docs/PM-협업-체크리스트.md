# PM 협업 체크리스트

> 뷰 제작(~03/12)과 실장 싱크업(03/13)까지 PM과 효율적으로 협업하기 위한 페이지별 확인 항목.
> 각 페이지의 와이어프레임/프로토타입을 공유하면서 아래 항목을 순서대로 확인한다.

---

## 작업 방식

```
1. 페이지별 와이어프레임 초안 작성
2. 아래 체크리스트로 PM에게 확인 요청
3. 피드백 즉시 반영 → 다음 페이지로 이동
4. 전체 완료 후 프로토타입 배포
```

**진행 순서**: 페이지 1 (대시보드) → 페이지 2 (상세) → 페이지 3 (튜터)

페이지 3은 페이지 1의 변형이므로, 1과 2가 확정되면 자연스럽게 결정된다.

---

## 페이지 1: 강의 제작 현황 대시보드

### 이미 확정된 것

- 오늘 진행 강의: 칸반 형태, 단계별(교안작성/촬영/편집/검수/완료) 구분
- 전체 프로젝트 리스트: 마감일(D-day) 기준 정렬
- 강의 제작 요청: 상단 우측 버튼 → 모달 폼

### 담당자 확인 필요

#### 오늘 진행 강의 (칸반)

- [ ] 칸반 컬럼 확정: `교안작성 | 리허설 | 촬영 | 편집 | 자막 | 검수` — 이 순서와 항목이 맞는지?
- [ ] 카드에 표시할 정보: 강의명, 튜터명, 챕터, 담당자 — 이 외에 더 필요한 정보?
- [ ] "오늘"의 기준: 오늘 시작일인 태스크? 오늘이 기간에 포함된 태스크? 오늘 마감인 태스크?
- [ ] 칸반 카드 클릭 시 동작: 바로 상세 페이지(페이지 2)로 이동? 아니면 해당 태스크로 스크롤?

#### 전체 프로젝트 리스트

- [ ] 리스트 vs 칸반: 어떤 형태가 기본? 둘 다 전환 가능하게?
- [ ] 카드/행에 표시할 정보: 강의명, 상태, 진척도, D-day, 담당자 — 우선순위 또는 추가 항목?
- [ ] 정렬 기준: D-day 순 기본? 다른 정렬 옵션 필요?
- [ ] 필터: 상태별, 사업부별, 담당자별 — 어떤 필터가 필요?
- [ ] 신호등(🟢🟠) 표시 필요한지?

#### 강의 제작 요청 폼

- [ ] 폼 필드 목록 확정 — 현재 노션 폼의 필드를 그대로 가져오면 되는지?
  - 강의명, 사용처(사업부), 제작유형, 트랙명, 롤아웃일, 챕터 수, 예상 분량, 튜터, 커리큘럼기획 매니저 ...
- [ ] 필수 필드 vs 선택 필드 구분
- [ ] 폼 제출 후 바로 프로젝트 상세로 이동? 리스트로 돌아가기?

> **담당자에게 요청**: 현재 사용 중인 노션 "강의 제작 일정 공유 폼" 캡처 또는 필드 목록

## 페이지 2: 강의 상세 페이지

### 이미 확정된 것

- 강의 정보 메타데이터 (튜터, 매니저, 편집자, 슬랙, 작업 링크 등)
- 챕터별 타임라인/간트
- 촬영 가이드 / 편집검수 가이드 바로가기
- PM 일정 조정 기능
- 세부 태스크 (챕터 단위, 상태 변경)

### 담당자 확인 필요

#### 강의 정보 영역

- [ ] 상단에 표시할 핵심 정보 우선순위: 어떤 정보를 가장 눈에 띄게?
- [ ] 작업 링크 목록 확정: 백오피스, 구글 드라이브, 교안, 커리큘럼 시트 — 이 외에 더 있는지?
- [ ] 슬랙 채널 링크: 클릭 시 슬랙으로 바로 이동하면 충분?

#### 일정 타임라인

- [ ] 타임라인 vs 간트차트 vs 테이블: PM이 선호하는 형태?
- [ ] 챕터 간 종속성 표시: 화살표로 연결해서 보여줄지?
- [ ] 지연된 태스크 시각적 표시 방식: 빨간색? 아이콘?

> **담당자에게 요청**: 현재 노션에서 일정을 관리하는 화면 캡처 (어떤 형태에 익숙한지 파악)

#### 가이드 문서

- [ ] 촬영 가이드 실제 내용/형식: 외부 링크(구글독스)? 시스템 내 페이지?
- [ ] 편집검수 가이드 실제 내용/형식: 같은 질문
- [ ] 가이드는 강의별로 다른지, 공통 템플릿인지?

> **담당자에게 요청**: 현재 사용 중인 촬영 가이드 / 편집검수 가이드 샘플

#### 세부 태스크

- [ ] 태스크 상태 변경 방식: 드롭다운? 버튼? 드래그?
- [ ] 태스크에 메모/코멘트 기능 필요?
- [ ] 태스크 담당자 변경: PM만 가능? 상세페이지에서 바로 변경?

#### 일정 조정

- [ ] 날짜 변경 시 후속 태스크 자동 조정 여부: 연쇄 이동? 수동 확인?
- [ ] 변경 이력 기록 필요?

- [ ] 튜터가 가장 먼저 봐야 하는 정보: "오늘 내가 할 일"이 맞는지?
- [ ] 튜터에게 알림/리마인드 표시가 필요한지? (예: "CH3 촬영 내일 마감")

# 페이지 1 상세 체크리스트 — 강의 제작 현황 대시보드

> 개발에 필요한 모든 의사결정 항목을 담당자 확인용으로 상세화한 문서.
> 와이어프레임 공유 시 이 체크리스트를 함께 검토한다.

---

## 1-A. 대시보드 전체 레이아웃

### 이미 확정된 것

- 상단: 오늘 진행 강의 (칸반)
- 하단: 전체 프로젝트 리스트
- 우측 상단: 강의 제작 요청 버튼

### 담당자 확인 필요

#### 화면 구성

- [ ] 오늘 진행 강의(칸반)와 전체 프로젝트 리스트의 비율: 50:50? 칸반이 더 크게? 접기/펼치기 가능?
- [ ] 대시보드 진입 시 기본 상태: 칸반이 펼쳐진 상태? 프로젝트 리스트가 먼저 보이는 상태?
- [ ] 네비게이션/사이드바 필요 여부: 대시보드 외 다른 페이지로 이동하는 메뉴 구조
- [ ] 상단에 요약 지표(숫자) 필요 여부: "진행 중 프로젝트 8건 / 오늘 태스크 12건 / 지연 2건" 같은 카운트

## 1-B. 오늘 진행 강의 (칸반)

### 이미 확정된 것

- 칸반(Kanban) 보드 형태
- 단계별 컬럼으로 오늘 강의 상태를 실시간 확인
- 핵심: "오늘 내가 뭘 챙겨야 하는지" 한눈에 파악

### 담당자 확인 필요

#### 컬럼 정의

- [ ] 칸반 컬럼(단계) 목록과 순서 최종 확정:
  - 현재 안: `교안작성 | 리허설 | 촬영 | 편집 | 자막 | 검수`
  - "완료" 컬럼 필요한지? (오늘 완료된 항목도 보여줄지)
  - "업로드" 단계도 포함하는지? (노션 데이터에 업로드 태스크 존재)
- [ ] 각 컬럼에 건수 표시: "촬영 (3)" 형태로 숫자 표시할지?
- [ ] 컬럼 간 드래그앤드롭으로 상태 변경 가능해야 하는지? (PM이 칸반에서 바로 상태 이동)

#### 카드 디자인

- [ ] 카드에 표시할 정보 (우선순위 순으로 확정):
  - 강의명 (필수)
  - 챕터 번호 (예: CH3)
  - 담당자명 (튜터? 편집자? 해당 단계 담당자?)
  - 마감일 또는 남은 일수
  - 기타: 슬랙 채널 바로가기? 진행 메모?
- [ ] 카드 색상/라벨: 사업부별 색상 구분? 지연 여부에 따른 색상?
- [ ] 같은 강의의 여러 챕터가 동시에 있을 때 그룹핑: 강의별로 묶어 표시? 개별 카드?

#### "오늘"의 정의

- [ ] "오늘 진행 강의"의 기준 확정:
  - (A) 오늘이 태스크 기간(시작일~종료일)에 포함되는 모든 태스크
  - (B) 오늘이 시작일인 태스크만
  - (C) 오늘이 종료일(마감)인 태스크만
  - (D) 현재 "진행" 상태인 모든 태스크 (날짜 무관)
- [ ] 오늘 해당 태스크가 없는 경우 빈 화면 처리: 빈 칸반? "오늘은 진행 중인 태스크가 없습니다" 메시지?

#### 카드 인터랙션

- [ ] 카드 클릭 시 동작:
  - (A) 해당 프로젝트의 상세 페이지(페이지 2)로 이동
  - (B) 해당 프로젝트 상세 페이지의 해당 태스크 위치로 바로 이동
  - (C) 카드 위에서 간단한 팝오버로 상세 정보 표시 → "상세 보기" 클릭 시 이동
- [ ] 카드에서 바로 상태 변경 가능한지? (예: "완료" 버튼)

## 1-C. 전체 프로젝트 리스트

### 이미 확정된 것

- 마감일(롤아웃) 기준 D-day가 핵심 정보
- 칸반 또는 리스트 형태 고려
- 프로젝트 상태, 진척도 요약 노출

### 담당자 확인 필요

#### 뷰 형태

- [ ] 기본 뷰: 리스트(테이블) vs 칸반 vs 카드 그리드 — 어떤 게 기본?
- [ ] 뷰 전환 기능: 리스트 ↔ 칸반 전환 토글 필요?
- [ ] 칸반으로 볼 경우 컬럼 기준: 프로젝트 상태별(기획/교안제작/촬영제작/리뷰/완료)?

#### 리스트(테이블) 뷰 — 컬럼 정의

표시할 정보를 우선순위 순으로 확정:

- [ ] **강의명**: `(v2.0) 강의제목` 형태 그대로?
- [ ] **프로젝트 상태**: 기획 / 교안 제작 / 촬영·제작 / 리뷰 / 완료 / 중단 — 뱃지? 텍스트?
- [ ] **진척도**: `12/25` 숫자? 프로그레스 바? 퍼센트?
- [ ] **D-day**: 롤아웃까지 남은 일수. 마감 초과 시 `D+N`으로 빨간색 표시?
- [ ] **담당자**: PM? 커리큘럼기획 매니저? 둘 다?
- [ ] **사용처(사업부)**: KDT / KDC / 신사업 표시 필요?
- [ ] **신호등**: 🟢🟠🔴 표시 필요? 자동 계산? 수동 설정?
- [ ] **제작 유형**: 신규 / 부분 리뉴얼 / 전체 리뉴얼 — 표시 필요?
- [ ] **튜터명**: 리스트에서 바로 보여야 하는지?
- [ ] 기타 추가 컬럼: 강의 지급일? 트랙명? 롤아웃 날짜(D-day 말고 실제 날짜)?

#### 정렬/필터/검색

- [ ] 기본 정렬: D-day 오름차순(마감 임박 순)이 맞는지?
- [ ] 정렬 옵션: 컬럼 클릭으로 정렬 전환 필요?
- [ ] 필터 항목 확정:
  - 프로젝트 상태별 (기획/교안제작/촬영제작/리뷰/완료/중단)
  - 사업부별 (KDT/KDC/신사업)
  - 담당자별
  - 제작 유형별
  - 기타?
- [ ] 검색: 강의명으로 검색 필요?
- [ ] "완료" 프로젝트 기본 노출 여부: 기본 숨김? 필터로 볼 수 있게?
- [ ] "중단" 프로젝트 기본 노출 여부: 같은 질문

#### 프로젝트 카드/행 인터랙션

- [ ] 클릭 시: 강의 상세 페이지(페이지 2)로 이동 — 이것만으로 충분?
- [ ] 리스트에서 바로 할 수 있어야 하는 액션: 상태 변경? 담당자 변경? 없음?
- [ ] 우클릭/더보기 메뉴: 프로젝트 삭제? 복제? 아카이브?

#### 빈 상태 / 페이지네이션

- [ ] 프로젝트가 0건일 때 화면: "강의 제작 요청으로 첫 프로젝트를 만들어보세요" 같은 안내?
- [ ] 프로젝트 수가 많을 때: 무한 스크롤? 페이지네이션? 현재 운영 중인 프로젝트 수는 보통 몇 건?

## 1-D. 강의 제작 요청 폼

### 이미 확정된 것

- 대시보드 상단 우측 "강의 제작 요청" 버튼

## 2-G. 강의 영상 피드백 탭

### 이미 확정된 것

- 상세 페이지의 세 번째 탭 (`영상 피드백`)
- 관리 단위: **강 단위** (1-1 = 1장 1강, 챕터 아래 최소 단위)
  - 강 수는 챕터별 예상 분량에서 자동 계산
- 플로우: 튜터 영상 업로드 → PM/커기매 검수(점수+코멘트) → 판정 → 피드백 전달
- 피드백란은 **튜터에게 비공개**
- 검수 역할 분리: PM(영상 품질) / 커기매(영상 내용)
- 점수: 5점 척도(1~5) + 텍스트 피드백(타임스탬프 포함)
- PM 6문항 + 커기매 6문항 (상세는 `docs/영상-피드백-상세-설계.md` 참조)
- 판정 로직:
  - **4.3점 이상**: 승인
  - **3.5 ~ 4.2점**: 보완 후 재검토
  - **3.4점 이하**: 재촬영 권장
  - 치명 문항(PM: 음성전달·촬영품질, 커기매: 학습목표·설명정확성) 2점 이하 시 평균 무관하게 보완/재촬영
- 질문별 점수와 코멘트 기반으로 **튜터 전달용 피드백 문구 자동 생성**
  - 총평 + 잘한 점(4~5점) + 보완 필요 사항(1~3점)
  - 텍스트 복사 → 슬랙 채널로 전달
- 복수 영상 지원: 한 강의 원본이 여러 영상일 경우 화살표 네비게이션 + 자동 재생
