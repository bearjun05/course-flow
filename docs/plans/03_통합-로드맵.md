## 문서 완결성 체크리스트

백엔드 개발자가 헷갈리지 않으려면 아래가 모두 있어야 함:

- [ ] 엔티티 필드가 타입/필수/기본값까지 명시
- [ ] 담당자 복수 배정: **배열**로 통일 (문서+코드)
- [ ] API Request/Response 스키마 (플랜 2)
- [ ] 상태 전이 다이어그램 (기획→교안→... 그림)
- [ ] D-Day 계산식 + 경계값 (`≤ 3` 노랑 등)
- [ ] 신호등: 수동 조작만, 자동 로직 없음 (정책 업데이트 필요)
- [ ] 권한 매트릭스 (역할 × 리소스 × 읽기/쓰기)
- [ ] Drive 폴더 규칙 + 파일명 정규식 (백엔드 할일로 이관됨)
- [ ] 슬랙 템플릿 + 복수 담당자 멘션 규칙
- [ ] 중단/재개 동작 (상태만 변경, 복구)
- [ ] 리뉴얼 시 기존 폴더 재활용 규칙
- [ ] 제작 요청 폼 필드 목록 (`제작요청-폼.md`)

## 주의

- 코드 수정과 문서 수정이 **항상 같이** 가야 함. 코드만 바뀌거나 문서만 바뀌면 불일치 발생.
- 백엔드 개발자 투입 전에 **프론트를 더 늘리지 않기**. 지금도 600줄+ 컴포넌트가 있어 정리가 먼저.
- 담당자 배열 마이그레이션은 API 스키마 확정 후 한 번에 진행.
