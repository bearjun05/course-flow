## 📅 작업 순서

### 1️⃣ 코드 퀄리티 정리 — **이번 주 진행**
→ [`01_코드-퀄리티-정리.md`](./01_코드-퀄리티-정리.md)

프론트 자체의 잘못된 동작과 정리되지 않은 부분을 고친다.
- 에듀웍스 WeeklyCalendar 읽기전용 누락 (버그)
- 담당자 파싱 로직 중복 제거
- 색상 상수 통합
- 큰 컴포넌트 분할
- 기획 모달 key 안정성

### 2️⃣ API 요청/응답 스키마 정의 — **이번 주~다음 주**
→ [`02_API-스키마-정의.md`](./02_API-스키마-정의.md)

백엔드 개발자가 API와 DB를 만들기 전에 **어떤 데이터가 오고 가는지**를 문서로 못 박는다.
- 엔티티별 필드 스키마 (Project / Task / Lecture / Chapter / User)
- API 요청/응답 예시
- 이 문서가 끝나야 DB 속성 리스트업 가능

### 3️⃣ 통합 로드맵 — **참고용**
→ [`03_통합-로드맵.md`](./03_통합-로드맵.md)

위 작업 이후 백엔드 개발자 투입까지의 전체 그림.
- 문서 완결성 체크리스트
- 백엔드 인수인계 순서
- 단계별 마일스톤
