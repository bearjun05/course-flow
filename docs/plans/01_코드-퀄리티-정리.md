## 🔴 High (즉시 수정 — 버그 또는 일관성 깨진 부분)

### H-1. 에듀웍스 WeeklyCalendar에 `readOnly` 전달 누락
- **위치**: `frontend/src/app/eduworks/[id]/page.tsx`의 `<WeeklyCalendar>` 호출부
- **문제**: WeeklyCalendar는 `onTaskToggle`만 조건부로 주지만, drag/편집 제어는 별도 `readOnly` prop이 없으면 막히지 않음. 검수자가 아닌 튜터/편집자도 태스크를 조작할 여지.
- **조치**:
  1. WeeklyCalendar에 `readOnly?: boolean` prop 추가 (MondayBoard처럼 drag/편집 전반을 차단)
  2. 에듀웍스에서 `readOnly` 전달, 단 `onTaskToggle`은 검수자일 때만 주입하는 현재 패턴 유지
- **검증**: 에듀웍스에서 튜터로 접속 → 이번 주 탭 → 검수 태스크가 아닌 항목 클릭해서 변경 안 되는지 확인

### H-2. 담당자 파싱 로직이 3곳에 중복
- **위치**:
  - `lib/utils.ts`의 `parseAssigneeNames` (기준)
  - `app/eduworks/[id]/page.tsx`에서 `reviewer.split(",")` 직접 사용
  - `components/detail/info-guide-tab.tsx`의 인라인 `parseAssignees`
- **조치**: 두 인라인 로직을 `parseAssigneeNames`로 통일
- **검증**: 기존 동작 동일, 쉼표 포함 문자열/공백 처리 일관

### H-3. planning-modal 장/강 `key={i}` 인덱스 남용
- **위치**: `components/detail/planning-modal.tsx`
- **문제**: 장/강을 중간에 삭제하면 인덱스가 밀리면서 input 포커스/상태가 이전 행에 들러붙을 수 있음
- **조치**: `DraftChapter`에 `id: string` 필드 추가 (클라이언트 생성 uuid 또는 timestamp), `key={ch.id}`로 교체. 강도 동일하게.
- **검증**: 장 3개 추가 → 중간 장 삭제 → 다른 장 input 값이 유지되는지 확인

## 🟡 Medium (유지보수성 / 확장성)

### M-1. 색상 상수 분산
- **위치**:
  - `summary-metrics.tsx` (RGB 직접)
  - `project-card.tsx` / `detail-header.tsx` / `info-guide-tab.tsx` (traffic light 4곳 재정의)
  - `monday-board.tsx` / `weekly-calendar.tsx` (GROUP_COLORS 별도)
- **조치**:
  1. `lib/constants.ts`에 `TRAFFIC_LIGHT_COLORS` 통합
  2. 파스텔 톤 챕터 색상은 `CHAPTER_COLORS_PASTEL` + `getChapterColorPastel()`로 명시적 분리
  3. summary-metrics의 RGB도 constants로 이동
- **검증**: 화면 색상 변화 없음. 한곳에서 수정 시 모든 곳에 반영되는지.

### M-2. `projects/[id]/page.tsx`의 useState 19개
- **위치**: `app/projects/[id]/page.tsx`
- **문제**: useMemo 의존성이 19개라 memoization 의미 없음. 상태 변경 시 불필요한 재렌더링.
- **조치**: `useReducer` 도입 또는 도메인별 그룹화(`useState`를 people / schedule / planning으로 묶음)
- **검증**: 동작 동일, React DevTools로 리렌더 횟수 감소 확인
- **주의**: 범위가 커서 마지막에 진행. 다른 리팩토링 끝난 후에.

### M-3. 600줄 이상 컴포넌트 분할
- **대상**:
  - `components/detail/work-status-tab.tsx` (612줄) → `DeliverableCell`, `ReviewCell`, `ApprovalCell`을 각 파일로 분리
  - `components/detail/monday-board.tsx` (968줄) → `TimelineGrid`, `TaskDragHandler`, `StageRow` 분리 (난이도 높음, 후순위)
- **조치**: 단계적으로 파일 분할. 각 셀 컴포넌트는 work-status-tab 내부에서만 쓰이므로 같은 폴더에 `_cells/` 서브폴더로 분리하는 방식 추천.
- **검증**: 기능 동일, import만 변경

### M-4. summary-metrics "오늘 태스크" 라벨 오해 소지
- **위치**: `components/dashboard/summary-metrics.tsx`
- **문제**: 실제로는 "진행 중인 모든 태스크" 수이지만 라벨이 "오늘 태스크". 문서/라벨 불일치
- **조치**: 둘 중 택일
  - (A) 라벨을 "진행 중 태스크"로 수정 (간단)
  - (B) 실제로 오늘 날짜에 걸친 태스크만 집계하도록 필터링 추가
- **제안**: (A) 먼저 적용하고 추후 논의
