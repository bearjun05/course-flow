## 1. MVP 기능 확정 기준

**MVP 목적**

- 강의 수가 증가하더라도 PM 리소스의 비례 증가 없이 온타임 딜리버리를 안정적으로 달성하고, 유관계자(커리큘럼기획 매니저, 튜터, 편집자/검수자)가 강의 제작 현황을 실시간으로 확인할 수 있게 한다.

| 문제 | 기대 성과 |
| --- | --- |
| PM 리소스가 강의 수에 비례 증가 | **주 7시간 절약** |
| 유관계자가 제작 현황을 실시간 확인 불가 | **실시간 확인 가능** |

**MVP 포함 여부 판단 기준**

1. PM 리소스 절약에 직접 기여하는가? — 수동 반복 작업을 시스템이 대체
2. 유관계자의 실시간 현황 확인을 가능하게 하는가? — 별도 문의 없이 시스템에서 직접 확인

위 두 기준 중 하나 이상을 충족하는 기능만 MVP에 포함

### 페이지 1: 강의 제작 현황 대시보드

| 영역 | 기능 | 설명 |
| --- | --- | --- |
| **오늘 진행 강의** | 칸반 보드 | 촬영·편집·검수·리허설·교안작성 단계별로 오늘 강의들의 실시간 상태 확인. "오늘 뭘 챙겨야 하는지" 한눈에 파악 |
| **전체 프로젝트 리스트** | 리스트/칸반 | 마감일(롤아웃) 기준 D-day, 프로젝트 상태, 진척도 요약 노출 |
| **강의 제작 요청** | 모달 폼 | 상단 오른쪽 "강의 제작 요청" 버튼 → 폼 작성 → 제출 시 프로젝트 리스트에 자동 추가 |

### 페이지 2: 강의 상세 페이지

| 영역 | 기능 | 설명 |
| --- | --- | --- |
| **강의 전체 일정** | 타임라인/간트 | 챕터별 촬영 → 편집 → 검수 일정 시각화 |
| **가이드 문서** | 바로가기 | 튜터용 촬영 가이드, 편집자/검수자용 편집·검수 가이드 |
| **일정 조정** | 인라인 수정 | PM이 일정을 직접 수정 가능 |
| **세부 태스크** | 챕터 단위 관리 | 챕터별 교안제작→촬영→편집→자막→검수 태스크, 담당자 지정, 상태 변경 |
| **강의 정보** | 메타데이터 | 튜터, 매니저, 편집자/검수자, 슬랙 채널, 작업 링크(백오피스·드라이브·교안), 예상 분량 |

### 페이지 3: 튜터 전용 대시보드

PM 대시보드(페이지 1)와 동일한 구조이나, 로그인한 튜터 본인이 담당한 프로젝트만 노출.

| 영역 | 기능 | PM 대시보드 대비 차이 |
| --- | --- | --- |
| **오늘 진행 강의** | 칸반 보드 | 본인 담당 강의만 필터링 |
| **내 프로젝트 리스트** | 리스트/칸반 | 본인 배정 프로젝트만 노출 |
| **강의 제작 요청** | — | 제거 (PM/커리큘럼기획 매니저 전용) |

카드 클릭 시 **페이지 2 (강의 상세 페이지)** 를 그대로 재사용한다.

### 역할별 권한 구분

| 기능 | PM | 튜터 |
| --- | --- | --- |
| 전체 프로젝트 조회 | O | X (본인 담당만) |
| 강의 제작 요청 | O | X |
| 일정 조정 | O | X |
| 본인 태스크 상태 변경 (교안 제작, 촬영) | O | O |
| 타인 태스크 상태 변경 (편집, 검수) | O | X (조회만) |
| 강의 정보 / 가이드 문서 열람 | O | O |

### 슬랙 연동 (MVP)

- 태스크 상태 변경 시 해당 슬랙 채널(`#콘텐츠_강의제작_강의명`)에 Webhook으로 알림 자동 발송
