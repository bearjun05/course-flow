### 영역 1. 매일 아침 상태 확인 + D-day 계산 ★ 최대 병목

> 진영님 문서: "촬영 타임라인 관리 페이지에서 현재 진행 중인 강의 목록 파악 (1분) → 각 강의별 제작 현황 트래킹 (1개당 약 10분)"

| 항목 | AS-IS | TO-BE |
|---|---|---|
| **작업** | 슬랙 채널 확인 → 드라이브 업로드 확인 → 제작 페이지 상태 변경 → D-day 계산 | 대시보드에서 지연/마감 태스크 자동 표시 |
| **빈도** | 매일 (주 5일) | — |
| **소요 시간** | **1분(목록 파악) + 10분×5개 = 51분/일** | 대시보드 확인 **5분/일** |
| **주간 절감** | — | **(51-5)×5 = 230분 → 약 3.8시간/주** |

**진영님 문서 기준 세부 내역:**

매일 오전 트래킹 (강의 1개당 ~10분):
1. 슬랙 제작 채널 이동 → 새로 업데이트된 내용 확인
2. 구글 드라이브 접속 → 새로 업로드된 내용 확인
   - 촬영 단계: 튜터가 원본 영상을 업로드했는지 (예: 5-5중 5-4까지만 → 플래그)
   - 편집 단계: 편집 담당자가 작업 파일을 업로드했는지
   - 자막 단계: 자막 담당자가 작업 파일을 업로드했는지
3. 제작 페이지 상태 변경 (장 > 단계 > 상태값)

업무 중간 대응:
- 슬랙 알림 확인 → 확인 후 종료 or 대응 필요 건 처리
- 드라이브 권한 요청, 촬영 방법 문의, 편집/검수 가이드 관련 문의 등

**시스템 대체:**
- 대시보드에서 전체 프로젝트 상태 한눈에 파악
- D-day 자동 카운팅 + 색상 경고 (red/yellow/red)
- 오늘 마감 태스크, 지연 태스크 자동 필터링
- 드라이브 업로드 상태 → 시스템에서 직접 확인 가능

### 영역 2. 커뮤니케이션 (F/U + 공지 + 리마인드)

> 진영님 문서에서 유형별 소요 시간이 명확하게 정리되어 있음

#### 2-1. 촬영/편집/검수 제작 현황 F/U

| 시점 | 수신자 | 내용 | 소요 시간 |
|---|---|---|---|
| 마감 하루 전 | 튜터 or 편집/검수 | "내일은 N장 촬영/편집/검수 전 날, 확인 후 드라이브 업로드" | **5~10분** |
| D-Day 오전 | 튜터 or 편집/검수 | "오늘은 N장 촬영 날, 마무리 후 드라이브 업로드" | **5~10분** |
| 완료 다음날 | 튜터 or 편집/검수 | "업로드 확인 완료, 다음 작업 이어 부탁" | **10분** |
| 딜레이 발생 | 튜터 or 편집/검수 | 현재 완료/진행 중/예정 작업 현황 + 완료 시점 문의 | **10~20분** |

→ 진영님 판단: **자동화 가능**

#### 2-2. 공지형 메시지

| 시점 | 수신자 | 내용 | 소요 시간 |
|---|---|---|---|
| 튜터 채널 초대 직후 | 튜터 | 환영 인사 + 담당자 소개 | **1분** |
| 촬영 1주일 전 | 튜터 | 촬영 운영 페이지 공유 + 회사 위치 | **5분** |
| 촬영 3일 전 | 튜터 | 방문 안내, 스크립트 작성 안내 | **5분** |
| 편집~검수자 초대 직후 | 편집~검수 | 일정 및 요청사항 안내 | **20분** |
| 롤아웃 완료 | 전체 | 수고 및 감사 인사 | **5분** |

→ 진영님 판단: **일부 자동화 가능** (편집~검수자 초대 직후는 강의마다 세부 요청사항이 달라 자동화 어려움)

- 촬영 중 문의, 스튜디오 이용 방법, 편집/검수 가이드 관련 문의
- → **자동화 불가** (판단 필요)

### 영역 4. 일정 딜레이 대응

> 진영님 문서: "현재 작업 현황 파악, 후속 일정 수정 및 대응까지 약 30분 이상 소요"

| 항목 | AS-IS | TO-BE |
|---|---|---|
| **작업** | 딜레이 감지 → 현황 파악 → 후속 일정 재산정 → 롤아웃 영향 판단 → 상위 보고 여부 결정 | 지연 자동 감지 + 연쇄 영향 시뮬레이션 |
| **빈도** | 주 1~2회 | — |
| **소요 시간** | 건당 **30분 이상** = **~60분/주** | 10분 (자동 감지 후 판단만) |
| **주간 절감** | — | **약 0.8시간/주** |

**진영님 문서 기준 딜레이 연쇄 시나리오:**
- 교안 딜레이 → 촬영 일정 후행 조정
- 촬영 딜레이 → 편집/자막/검수 **전 단계 연쇄 지연**
- 검수 딜레이 → 롤아웃 직전 단계이므로 **롤아웃 영향 가능성 가장 높음**

**롤아웃 영향 판단 기준:**
- 영향 없음: 각 단계 기본 소요 기간 내 흡수 가능 → PM 자체 일정 재조정
- 영향 있음: 누적 지연이 완충 초과 → **상위 결재 통해 롤아웃 날짜 변경** (PM 단독 불가)

**시스템 대체:**
- 지연 태스크 자동 감지 + 후속 태스크 영향 자동 계산
- "롤아웃 영향 여부" 자동 판별 → 상위 보고 필요 시 알림

## 핵심 메시지 (미팅용)

### 리소스가 가장 많이 들어가는 TOP 3

1. **매일 아침 상태 확인 + D-day 계산** — 주 4.25시간 (전체의 46%)
   - 5개 프로젝트 × 1개당 10분 = 매일 51분
   - 슬랙 → 드라이브 → 제작 페이지 상태 변경 → D-day 확인 반복
   - → 대시보드 + 자동 D-day로 **3.8시간 절감**

2. **커뮤니케이션 (F/U + 리마인드)** — 주 2시간 (22%)
   - 마감 전/당일/딜레이 시 슬랙 메시지 수동 작성 (건당 5~20분)
   - 튜터 안내, 편집~검수자 온보딩 공지
   - → 자동 알림 + 템플릿으로 **1.5시간 절감**

3. **신호등(건강도) 수동 판단** — 주 1.25시간 (14%)
   - 프로젝트별 종합 상태를 머릿속으로 판단 → 수동 색상 변경
   - → 지연 태스크 기반 자동 건강도 계산으로 **1.25시간 절감**

### 한 줄 요약

> 현재 PM이 **주 9.25시간**을 수동 관리에 사용하고 있으며, CourseFlow 도입으로 **주 7~8시간을 시스템이 대체**한다. 특히 "상태 확인 + 커뮤니케이션"이 전체의 68%를 차지하며, 대시보드 + 자동 알림만으로 5.3시간을 절감할 수 있다.

### 강 단위 영상 업로드 정책

> "하나의 강 당 원본 영상 개수는 사전에 미리 지정하지 못함 → 강 단위로 올리고 완료 버튼을 누르는 형식"
> "1-1을 이루는 영상이 1개일수도, 5개일수도 → 2개 이상이면 편집 단계에서 합본"

→ 시스템 설계: 강 단위 "업로드 완료" 액션 필요 (자동 카운트 X, 명시적 완료 처리)

| 커뮤니케이션 (F/U) | **태스크 수에 비례** → 프로젝트 수 × 평균 태스크 수 | 프로젝트당 평균 주 2건 F/U |
| 일정 딜레이 대응 | **프로젝트 수에 비례 + 확률 증가** | 프로젝트 많을수록 딜레이 동시 발생 확률 ↑ |
| 신호등 판단 | **프로젝트 수에 정비례** | 프로젝트당 5분 × 주 3회 |
| 일정 산정 | **신규 프로젝트 수에 비례** | 10개 동시관리 시 월 3~4건 신규 |
| 환경 세팅 | **신규 프로젝트 수에 비례** | 위와 동일 |
| 공지형 메시지 | **신규 프로젝트 수에 비례** | 프로젝트 시작 시 집중 발생 |

### 영역별 상세 시뮬레이션

#### 영역 1. 매일 상태 확인 + D-day 계산

```
현재 (5개): 1분(목록) + 10분 × 5개 = 51분/일 → 255분/주 (4.25h)
10개 관리:  1분(목록) + 10분 × 10개 = 101분/일 → 505분/주 (8.4h)
```

> 이것만으로 이미 하루 업무 시간(8h)의 21%를 매일 아침 소비

#### 영역 2. 커뮤니케이션

```
현재 (5개): F/U 주 10건 × 평균 10분 = 100분/주
            공지형 주 2건 × 평균 8분 = 16분/주
            합계: 116분/주 (1.9h)

10개 관리:  F/U 주 20건 × 평균 10분 = 200분/주
            공지형 주 4건 × 평균 8분 = 32분/주
            합계: 232분/주 (3.9h)
```

> F/U 건수가 2배 → 하루 중 슬랙 메시지 작성에만 40분 이상 소요

#### 영역 3. 일정 딜레이 대응

```
현재 (5개): 주 1~2회 × 30분 = 45~60분/주 (1h)
10개 관리:  주 3~4회 × 30분 = 90~120분/주 (1.75h)
```

> 프로젝트 수 2배 → 딜레이 발생 빈도 2배 이상 (동시 딜레이 시 복잡도 급증)
> 딜레이 2건이 겹치면 일정 재조정이 서로 영향 → 건당 소요 시간도 증가

#### 영역 4. 신호등 판단

```
현재 (5개): 5분 × 5개 × 주 3회 = 75분/주 (1.25h)
10개 관리:  5분 × 10개 × 주 3회 = 150분/주 (2.5h)
```

#### 영역 5~6. 일정 산정 + 환경 세팅

```
현재 (5개 관리, 월 2건 신규): (75분 + 37분) × 2건/4주 = 56분/주 (0.9h)
10개 관리 (월 3~4건 신규):    (75분 + 37분) × 3.5건/4주 = 98분/주 (1.6h)
```

> 10개 동시 관리를 유지하려면 신규 프로젝트 유입도 늘어남
